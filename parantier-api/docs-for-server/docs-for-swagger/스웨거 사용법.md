# Swagger UI 사용법

## 접속 URL
- Swagger UI: http://localhost:8080/swagger-ui.html
- API 문서 (JSON): http://localhost:8080/v3/api-docs

## 사용 방법

### 1. API 테스트하기
1. Swagger UI 접속
2. 테스트할 API 엔드포인트 선택 (예: POST /api/auth/signup)
3. "Try it out" 버튼 클릭
4. Request body 입력
5. "Execute" 버튼 클릭
6. Response 확인

### 2. 회원가입 API 테스트 예시
```json
{
  "email": "test@example.com",
  "password": "password123!",
  "username": "홍길동"
}
```

## Controller에서 Swagger 어노테이션 사용법

### @Tag - API 그룹화
```java
@Tag(name = "인증", description = "회원가입, 로그인 등 인증 관련 API")
@RestController
@RequestMapping("/api/auth")
public class AuthController { }
```

### @Operation - API 설명
```java
@Operation(summary = "회원가입", description = "새로운 사용자를 등록합니다.")
@PostMapping("/signup")
public ResponseEntity<SignupResponse> signup(...) { }
```

### @Schema - DTO 필드 설명
```java
public class SignupRequest {
    @Schema(description = "이메일", example = "user@example.com")
    private String email;

    @Schema(description = "비밀번호", example = "password123!")
    private String password;
}
```

## 설정 파일 위치
- SpringDoc 설정: src/main/resources/application.yml
- OpenAPI 커스텀 설정: src/main/java/com/mapo/palantier/config/OpenApiConfig.java
- Controller 예시: src/main/java/com/mapo/palantier/user/presentation/AuthController.java

## 주의사항
- 개발 환경에서만 인증 없이 접근 가능 (SecurityConfig에서 설정됨)
- 프로덕션 환경에서는 비활성화하거나 인증 필요
- 민감한 정보(비밀번호 등)는 응답에 포함하지 말 것
