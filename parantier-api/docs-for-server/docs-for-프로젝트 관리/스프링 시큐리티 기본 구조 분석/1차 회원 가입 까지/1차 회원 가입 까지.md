## Spring Security 설정 상세 (SecurityConfig.java)

### 1. PasswordEncoder Bean
```java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```
**역할:**
- 비밀번호 암호화/검증에 사용
- BCrypt 알고리즘 (단방향 해시)
- 회원가입 시 비밀번호 암호화
- 로그인 시 비밀번호 검증 (다음 단계)

### 2. SecurityFilterChain 설정

#### CSRF 비활성화
```java
.csrf(csrf -> csrf.disable())
```
- REST API는 stateless하므로 CSRF 불필요
- JWT 토큰 기반 인증에서는 필요 없음

#### 세션 정책: STATELESS
```java
.sessionManagement(session -> session
    .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
)
```
- 서버에 세션을 저장하지 않음
- 모든 요청은 독립적 (JWT 토큰으로 인증)

#### 인증 예외 경로 설정
```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/actuator/health", "/actuator/health/**").permitAll()
    .requestMatchers("/swagger-ui/**", "/v3/api-docs/**", "/swagger-ui.html").permitAll()
    .requestMatchers("/api/**").permitAll()  // 현재는 전체 허용
    .anyRequest().authenticated()
)
```

**permitAll() 경로:**
- `/actuator/health`: 헬스체크 (인증 불필요)
- `/swagger-ui/**`: Swagger UI (개발 편의)
- `/api/**`: 현재는 전체 허용 (JWT 추가 후 수정 예정)

## 회원가입 플로우

```
1. Client → POST /api/auth/signup
   {
     "email": "user@example.com",
     "password": "password123!",
     "username": "홍길동"
   }

2. AuthController.signup()
   ↓
3. AuthService.signup()
   - 이메일 중복 체크 (UserRepository.existsByEmail)
   - 비밀번호 암호화 (PasswordEncoder.encode)
   - User 도메인 생성 (User.createNewUser)
   - 저장 (UserRepository.save)
   ↓
4. UserRepositoryImpl → UserMapper → DB
   ↓
5. Response 201 Created
   {
     "id": 1,
     "email": "user@example.com",
     "username": "홍길동",
     "role": "ROLE_USER",
     "createdAt": "2026-03-21T20:56:30.706Z"
   }
```

## 현재 Security 동작 방식

### 1. 인증되지 않은 상태
- 모든 `/api/**` 경로에 접근 가능 (permitAll)
- Spring Security 필터는 통과하지만 인증 체크 안 함

### 2. 비밀번호 암호화
- 회원가입 시: `passwordEncoder.encode(rawPassword)`
- DB 저장: BCrypt 해시값 저장 (예: `$2a$10$...`)

### 3. 아직 구현 안 된 부분
- [ ] JWT 토큰 생성
- [ ] JWT 토큰 검증
- [ ] JWT 인증 필터
- [ ] 로그인 API
- [ ] 인증이 필요한 API 보호

## 다음 단계: JWT 인증 구현

### 추가될 파일
```
config/
└── JwtConfig.java           # JWT 설정 (secret, expiration)

user/application/
├── JwtTokenProvider.java    # JWT 생성/검증
└── AuthService.java         # login() 메서드 추가

user/presentation/dto/
├── LoginRequest.java        # 로그인 요청
└── LoginResponse.java       # 로그인 응답 (JWT 포함)

shared/security/
└── JwtAuthenticationFilter.java  # JWT 검증 필터
```

### SecurityConfig 변경 예정
```java
.addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class)
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/auth/**").permitAll()  // 회원가입/로그인만 허용
    .requestMatchers("/api/**").authenticated()   // 나머지는 인증 필요
)
```

## 참고 사항

### BCrypt 비밀번호 검증 방식
```java
// 저장된 해시: $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
// 평문 비밀번호: password123!

passwordEncoder.matches("password123!", storedHash)  // true
passwordEncoder.matches("wrongpassword", storedHash) // false
```

### 보안 고려사항
- 비밀번호는 절대 평문으로 저장하지 않음
- 응답(Response)에 비밀번호 포함하지 않음
- 현재는 `/api/**` 전체 허용이지만, JWT 추가 후 최소 권한 원칙 적용
