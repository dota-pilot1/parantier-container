# 회원 가입 구현 계획 (DDD 기반)

## DDD 패키지 구조
```
com.mapo.palantier.user/
├── domain/           # 도메인 계층
│   ├── User.java
│   ├── UserRole.java
│   └── UserRepository.java (인터페이스)
├── infrastructure/   # 인프라 계층 (MyBatis 구현)
│   ├── UserRepositoryImpl.java
│   └── UserMapper.java
├── application/      # 애플리케이션 계층
│   └── AuthService.java
└── presentation/     # 프레젠테이션 계층
    ├── AuthController.java
    └── dto/
        ├── SignupRequest.java
        └── SignupResponse.java
```

## 구현 순서

### 1. DB 스키마 ✅
- [완료] users 테이블 생성 (DBeaver에서 수동 실행)
- 위치: src/main/resources/db/migration/V1__create_users_table.sql
- 컬럼: id, email, password, username, role, is_active, created_at, updated_at

### 2. Domain 계층 (도메인 로직) ✅
- [완료] User.java (Aggregate Root)
  - createNewUser() 팩토리 메서드
  - deactivate(), activate(), isAdmin() 도메인 로직
- [완료] UserRole.java (Value Object - Enum)
  - ROLE_USER, ROLE_ADMIN
- [완료] UserRepository.java (Repository Interface)
  - save(), findByEmail(), existsByEmail()

### 3. Infrastructure 계층 (기술 구현) ✅
- [완료] UserMapper.java (@Mapper 인터페이스)
- [완료] UserMapper.xml (MyBatis SQL)
  - insert (useGeneratedKeys로 ID 자동 생성)
  - findByEmail
  - existsByEmail
- [완료] UserRepositoryImpl.java
  - UserRepository 인터페이스 구현
  - UserMapper 호출

### 4. Application 계층 (유스케이스) ✅
- [완료] PasswordEncoder Bean 설정 (SecurityConfig.java)
- [완료] AuthService.java
  - signup() 메서드
  - 이메일 중복 체크 (IllegalArgumentException)
  - 비밀번호 BCrypt 암호화
  - UserRepository를 통한 저장

### 5. Presentation 계층 (API) ✅
- [완료] SignupRequest.java (입력 DTO)
  - @Schema 어노테이션으로 Swagger 문서화
- [완료] SignupResponse.java (출력 DTO)
  - from() 정적 팩토리 메서드
  - 비밀번호 제외하고 응답
- [완료] AuthController.java
  - POST /api/auth/signup
  - HTTP 201 Created 응답

### 6. 보안 설정 ✅
- [완료] /api/auth/** permitAll 설정 (SecurityConfig.java)
- [완료] BCryptPasswordEncoder Bean 등록

### 7. MyBatis 리소스 ✅
- [완료] src/main/resources/mybatis/mapper/UserMapper.xml
- ResultMap 설정 (snake_case → camelCase 자동 변환)
