# 로그인 구현 (JWT)

## 개요
회원가입 이후, 로그인 기능을 JWT(JSON Web Token) 기반으로 구현했습니다.

## 구현 내용

### 1. JWT 라이브러리 추가 (build.gradle)
```gradle
// JWT
implementation 'io.jsonwebtoken:jjwt-api:0.12.6'
runtimeOnly 'io.jsonwebtoken:jjwt-impl:0.12.6'
runtimeOnly 'io.jsonwebtoken:jjwt-jackson:0.12.6'
```

### 2. JWT 설정 (application.yml)
```yaml
# JWT Configuration
jwt:
  secret: your-256-bit-secret-key-change-this-in-production-please-make-it-long-enough
  expiration: 86400000  # 24시간 (밀리초)
```

### 3. JwtTokenProvider 구현
**위치**: `com.mapo.palantier.config.JwtTokenProvider`

**주요 메서드**:
- `generateToken(email, role)`: JWT 토큰 생성
- `getEmailFromToken(token)`: 토큰에서 이메일 추출
- `getRoleFromToken(token)`: 토큰에서 역할 추출
- `validateToken(token)`: 토큰 유효성 검증

### 4. DTO 작성
**LoginRequest** (`user.presentation.dto.LoginRequest`)
```java
- email: String
- password: String
```

**LoginResponse** (`user.presentation.dto.LoginResponse`)
```java
- accessToken: String (JWT 토큰)
- tokenType: String (Bearer)
- email: String
- username: String
- role: String
```

### 5. AuthService 로그인 메서드
**위치**: `user.application.AuthService.login()`

**로그인 프로세스**:
1. 이메일로 사용자 조회
2. 비밀번호 검증 (BCrypt)
3. 계정 활성화 여부 확인
4. JWT 토큰 생성 및 반환

### 6. AuthController 로그인 API
**엔드포인트**: `POST /api/auth/login`

**Request Body**:
```json
{
  "email": "terecal@daum.net",
  "password": "password123!"
}
```

**Response**:
```json
{
  "accessToken": "eyJhbGciOiJIUzUxMiJ9...",
  "tokenType": "Bearer",
  "email": "terecal@daum.net",
  "username": "오현석",
  "role": "ROLE_USER"
}
```

### 7. JwtAuthenticationFilter 구현
**위치**: `com.mapo.palantier.config.JwtAuthenticationFilter`

**필터 동작**:
1. Request Header에서 Authorization Bearer 토큰 추출
2. 토큰 유효성 검증
3. 토큰에서 사용자 정보 추출 (email, role)
4. Spring Security 인증 객체 생성
5. SecurityContext에 인증 정보 설정

### 8. SecurityConfig 업데이트
**변경 사항**:
- JWT 인증 필터를 UsernamePasswordAuthenticationFilter 앞에 추가
- `/api/auth/signup`, `/api/auth/login`은 인증 없이 접근 가능
- 나머지 `/api/**` 엔드포인트는 인증 필요

```java
.addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class)
```

## 테스트 결과

### 로그인 성공
```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "terecal@daum.net", "password": "password123!"}'
```

**응답**:
```json

  "username": "오현석",
  "role": "ROLE_USER"

```

### JWT 인증 테스트
```bash
curl -X GET http://localhost:8080/actuator/info \
  -H "Authorization: Bearer {accessToken}"
```
→ 인증 성공

## 보안 고려사항
1. **JWT Secret Key**: 프로덕션에서는 환경변수로 관리 필요
2. **Token Expiration**: 24시간으로 설정 (필요시 조정 가능)
3. **HTTPS**: 프로덕션에서는 HTTPS 필수
4. **Refresh Token**: 향후 구현 검토 필요
