jwt 토큰의 가장 기본적인 로직:
로그인을 통과 한 사람에게 토큰을 발행
그 토큰을 가진 사람이 다른 요청에 대해 토큰을 같이 보내면 로그인한 사람으로 간주

jwt 토큰 만료에 대비한 리프레시트 토큰 발급 프로세스 기본:
✅ 로그인 시에 엑세스 토큰과 리프레시 토큰을 같이 발급
✅ 유저가 보낸 액세스 토큰의 기한이 만료된 경우 → 401 Unauthorized 응답
✅ 클라이언트가 401 받으면 → 별도로 /api/auth/refresh 엔드포인트에 리프레시 토큰만 보냄
✅ 서버가 리프레시 토큰의 만료 기간 확인
✅ 리프레시 토큰이 유효하면 → 새 액세스 토큰 재발급
✅ 리프레시 토큰도 만료되었으면 → 401 응답, 재로그인 필요

구현 관점 (세부 동작) - 제 설명 ✅
1. 액세스 토큰 만료 → 401 응답
2. (별도 요청) 클라이언트가 /refresh에 리프레시 토큰 보냄
3. 서버가 리프레시 토큰 만료 체크
4. 유효하면 재발급 / 만료됐으면 401


리프레시 토큰 DB 저장 이유:
- 로그아웃 시 토큰 무효화 가능
- 탈취 의심 시 강제 무효화 가능
- 비밀번호 변경 시 모든 기기 로그아웃 가능
