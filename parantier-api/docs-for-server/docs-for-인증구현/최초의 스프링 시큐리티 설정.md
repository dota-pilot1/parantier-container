# Spring Security 기본 설정

## 현재 설정 (SecurityConfig.java)

### 1. CSRF 비활성화
```java
.csrf(csrf -> csrf.disable())
```
- REST API는 stateless하므로 CSRF 토큰 불필요
- JWT 기반 인증에서는 CSRF 공격 위험 낮음

### 2. 세션 정책: STATELESS
```java
.sessionManagement(session -> session
    .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
)
```
- 서버에 세션 저장하지 않음
- JWT 토큰 기반 인증 준비

### 3. 인증 없이 접근 가능한 엔드포인트
```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/actuator/health", "/actuator/health/**").permitAll()
    .requestMatchers("/swagger-ui/**", "/v3/api-docs/**", "/swagger-ui.html").permitAll()
    .requestMatchers("/api/**").permitAll()  // 현재는 전체 허용
)
```

#### permitAll() 설정된 경로:
- `/actuator/health`: 헬스체크 (K8s 등에서 사용)
- `/swagger-ui/**`: Swagger UI 접근
- `/api/**`: 현재는 모든 API 허용 (추후 JWT 인증 추가 예정)

### 4. PasswordEncoder 설정
```java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```
- BCrypt 알고리즘 사용 (비밀번호 암호화)
- 회원가입 시 비밀번호 해싱에 사용
