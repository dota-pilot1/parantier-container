# 전역 예외 처리 시스템 파일 구조

## 디렉토리 트리

```
src/main/java/com/mapo/palantier/
├── common/
│   ├── dto/
│   │   └── ErrorResponse.java                    # 에러 응답 DTO
│   │
│   └── exception/
│       ├── ErrorCode.java                         # 에러 코드 Enum (중앙 관리)
│       ├── GlobalExceptionHandler.java            # 전역 예외 처리기
│       │
│       ├── DuplicateEmailException.java           # 이메일 중복 예외 (409)
│       ├── AuthenticationException.java           # 인증 실패 예외 (401)
│       ├── AccountInactiveException.java          # 계정 비활성화 예외 (403)
│       └── UserNotFoundException.java             # 사용자 없음 예외 (404)
│
└── user/
    └── application/
        └── AuthService.java                       # 비즈니스 로직에서 예외 사용
```

## 파일별 상세 설명

### 1. ErrorCode.java (에러 코드 중앙 관리)

**경로**: `src/main/java/com/mapo/palantier/common/exception/ErrorCode.java`

**역할**:
- 모든 에러 코드를 Enum으로 중앙 관리
- 에러 코드, 메시지, HTTP 상태 코드를 함께 관리
- 매직 스트링 제거 및 컴파일 타임 안전성 확보

**주요 내용**:
```java
public enum ErrorCode {
    // 인증 관련 (401)
    INVALID_CREDENTIALS("INVALID_CREDENTIALS", "이메일 또는 비밀번호가 일치하지 않습니다", HttpStatus.UNAUTHORIZED),

    // 권한 관련 (403)
    ACCOUNT_INACTIVE("ACCOUNT_INACTIVE", "비활성화된 계정입니다", HttpStatus.FORBIDDEN),

    // 리소스 관련 (404)
    USER_NOT_FOUND("USER_NOT_FOUND", "사용자를 찾을 수 없습니다", HttpStatus.NOT_FOUND),

    // 중복 관련 (409)
    DUPLICATE_EMAIL("DUPLICATE_EMAIL", "이미 사용 중인 이메일입니다", HttpStatus.CONFLICT),
    
    // ...

    private final String code;
    private final String message;
    private final HttpStatus httpStatus;
}
```

**의존성**: 없음 (다른 파일들이 이 파일을 참조)

### 2. ErrorResponse.java (에러 응답 DTO)

**경로**: `src/main/java/com/mapo/palantier/common/dto/ErrorResponse.java`

**역할**:
- 클라이언트에게 반환할 에러 응답의 통일된 형식 정의
- JSON 직렬화를 위한 DTO

**주요 내용**:
```java
@Getter
@Builder
@AllArgsConstructor
public class ErrorResponse {
    private String errorCode;         // 예: "DUPLICATE_EMAIL"
    private String message;            // 예: "이미 사용 중인 이메일입니다"
    private LocalDateTime timestamp;   // 에러 발생 시각
    private String path;               // API 경로

    public static ErrorResponse of(String errorCode, String message, String path) {
        return ErrorResponse.builder()
                .errorCode(errorCode)
                .message(message)
                .timestamp(LocalDateTime.now())
                .path(path)
                .build();
    }
}
```

**의존성**: 없음

**응답 예시**:
```json
{
  "errorCode": "DUPLICATE_EMAIL",
  "message": "이미 사용 중인 이메일입니다",
  "timestamp": "2026-03-22T14:30:00",
  "path": "/api/auth/signup"
}
```

### 3. 커스텀 예외 클래스들

모든 커스텀 예외는 공통 구조를 따릅니다:

```java
@Getter
public class XxxException extends RuntimeException {
    private final ErrorCode errorCode;

    public XxxException(ErrorCode errorCode) {
        super(errorCode.getMessage());
        this.errorCode = errorCode;
    }

    public XxxException(ErrorCode errorCode, Throwable cause) {
        super(errorCode.getMessage(), cause);
        this.errorCode = errorCode;
    }
}
```

#### 3.1 DuplicateEmailException.java

**역할**: 이메일 중복 시 발생
**HTTP 상태**: 409 Conflict
**의존성**: ErrorCode.java

**사용 위치**: `AuthService.signup()`

**사용 예시**:
```java
if (userRepository.existsByEmail(email)) {
    throw new DuplicateEmailException(ErrorCode.DUPLICATE_EMAIL);
}
```

#### 3.2 AuthenticationException.java

**역할**: 인증 실패 시 발생 (이메일 없음, 비밀번호 불일치)
**HTTP 상태**: 401 Unauthorized
**의존성**: ErrorCode.java

**사용 위치**: `AuthService.login()`

**사용 예시**:
```java
User user = userRepository.findByEmail(email)
        .orElseThrow(() -> new AuthenticationException(ErrorCode.INVALID_CREDENTIALS));

if (!passwordEncoder.matches(password, user.getPassword())) {
    throw new AuthenticationException(ErrorCode.INVALID_CREDENTIALS);
}
```

#### 3.3 AccountInactiveException.java

**역할**: 비활성화된 계정 접근 시 발생
**HTTP 상태**: 403 Forbidden
**의존성**: ErrorCode.java

**사용 위치**: `AuthService.login()`

**사용 예시**:
```java
if (!user.getIsActive()) {
    throw new AccountInactiveException(ErrorCode.ACCOUNT_INACTIVE);
}
```

#### 3.4 UserNotFoundException.java

**역할**: 사용자를 찾을 수 없을 때 발생
**HTTP 상태**: 404 Not Found
**의존성**: ErrorCode.java

**사용 위치**: `AuthService.getUserByEmail()`

**사용 예시**:
```java
return userRepository.findByEmail(email)
        .orElseThrow(() -> new UserNotFoundException(ErrorCode.USER_NOT_FOUND));
```

### 4. GlobalExceptionHandler.java (전역 예외 처리기)

**경로**: `src/main/java/com/mapo/palantier/common/exception/GlobalExceptionHandler.java`

**역할**:
- 애플리케이션 전역에서 발생하는 모든 예외를 한 곳에서 처리
- 각 예외 타입별로 적절한 HTTP 상태 코드와 ErrorResponse 반환
- @RestControllerAdvice로 전역 예외 처리 활성화

**의존성**:
- ErrorResponse.java
- DuplicateEmailException.java
- AuthenticationException.java
- AccountInactiveException.java
- UserNotFoundException.java

**주요 구조**:
```java
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(DuplicateEmailException.class)
    public ResponseEntity<ErrorResponse> handleDuplicateEmailException(...) {
        ErrorResponse errorResponse = ErrorResponse.of(
                e.getErrorCode().getCode(),
                e.getMessage(),
                request.getRequestURI()
        );
        return ResponseEntity.status(e.getErrorCode().getHttpStatus()).body(errorResponse);
    }

    @ExceptionHandler(AuthenticationException.class)
    public ResponseEntity<ErrorResponse> handleAuthenticationException(...) { ... }

    @ExceptionHandler(AccountInactiveException.class)
    public ResponseEntity<ErrorResponse> handleAccountInactiveException(...) { ... }

    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleUserNotFoundException(...) { ... }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleException(...) {
        // 모든 예상하지 못한 예외 처리
    }
}
```

**처리 흐름**:
1. Controller나 Service에서 예외 발생
2. Spring이 자동으로 @RestControllerAdvice 클래스 탐색
3. 발생한 예외 타입과 매칭되는 @ExceptionHandler 메서드 실행
4. ErrorResponse 객체 생성 및 반환

### 5. AuthService.java (비즈니스 로직)

**경로**: `src/main/java/com/mapo/palantier/user/application/AuthService.java`

**역할**: 인증/인가 관련 비즈니스 로직 처리

**의존성**:
- DuplicateEmailException.java
- AuthenticationException.java
- AccountInactiveException.java
- UserNotFoundException.java
- ErrorCode.java

**사용 예시**:
```java
@Service
@RequiredArgsConstructor
public class AuthService {

    @Transactional
    public User signup(String email, String password, String username) {
        if (userRepository.existsByEmail(email)) {
            throw new DuplicateEmailException(ErrorCode.DUPLICATE_EMAIL);
        }
        // ...
    }

    @Transactional(readOnly = true)
    public String login(String email, String password) {
        User user = userRepository.findByEmail(email)
                .orElseThrow(() -> new AuthenticationException(ErrorCode.INVALID_CREDENTIALS));

        if (!passwordEncoder.matches(password, user.getPassword())) {
            throw new AuthenticationException(ErrorCode.INVALID_CREDENTIALS);
        }

        if (!user.getIsActive()) {
            throw new AccountInactiveException(ErrorCode.ACCOUNT_INACTIVE);
        }
        // ...
    }

    @Transactional(readOnly = true)
    public User getUserByEmail(String email) {
        return userRepository.findByEmail(email)
                .orElseThrow(() -> new UserNotFoundException(ErrorCode.USER_NOT_FOUND));
    }
}
```

## 파일 간 의존성 다이어그램

```
ErrorCode.java (에러 코드 정의)
    ↑
    ├─── DuplicateEmailException.java ─────┐
    ├─── AuthenticationException.java ─────┤
    ├─── AccountInactiveException.java ────┤
    └─── UserNotFoundException.java ───────┤
                                           │
ErrorResponse.java (에러 응답 DTO)        │
    ↑                                      │
    │                                      │
    └──────────────────────────────────────┤
                                           │
                                           ↓
                        GlobalExceptionHandler.java
                        (전역 예외 처리기)
                                           ↑
                                           │
                        AuthService.java (비즈니스 로직)
                        - signup()
                        - login()
                        - getUserByEmail()
```

**의존성 설명**:
1. **ErrorCode.java**: 최상위, 모든 커스텀 예외가 이를 참조
2. **커스텀 예외들**: ErrorCode를 사용
3. **GlobalExceptionHandler**: ErrorResponse와 모든 커스텀 예외를 사용
4. **AuthService**: 커스텀 예외와 ErrorCode를 사용

## HTTP 상태 코드별 파일 매핑

| HTTP 상태 | 코드 | Exception 파일                    | ErrorCode               |
|-----------|------|-----------------------------------|-------------------------|
| 400       | Bad Request | (IllegalArgumentException)  | -                       |
| 401       | Unauthorized | AuthenticationException.java  | INVALID_CREDENTIALS     |
| 403       | Forbidden | AccountInactiveException.java    | ACCOUNT_INACTIVE        |
| 404       | Not Found | UserNotFoundException.java       | USER_NOT_FOUND          |
| 409       | Conflict | DuplicateEmailException.java     | DUPLICATE_EMAIL         |
| 500       | Internal Server Error | (Exception)    | INTERNAL_SERVER_ERROR   |

## 새로운 예외 추가 시 체크리스트

새로운 예외를 추가하려면 다음 파일들을 수정/추가해야 합니다:

### 1. ErrorCode.java에 새 에러 코드 추가
```java
// 예: 비밀번호 형식 오류 추가
INVALID_PASSWORD_FORMAT("INVALID_PASSWORD_FORMAT", "비밀번호는 8자 이상이어야 합니다", HttpStatus.BAD_REQUEST)
```

### 2. 새 Exception 클래스 생성
```
src/main/java/com/mapo/palantier/common/exception/InvalidPasswordFormatException.java
```

### 3. GlobalExceptionHandler.java에 핸들러 추가
```java
@ExceptionHandler(InvalidPasswordFormatException.class)
public ResponseEntity<ErrorResponse> handleInvalidPasswordFormatException(...) {
    // ...
}
```

### 4. 비즈니스 로직에서 사용
```java
if (!isValidPasswordFormat(password)) {
    throw new InvalidPasswordFormatException(ErrorCode.INVALID_PASSWORD_FORMAT);
}
```
