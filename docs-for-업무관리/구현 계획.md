# 업무 관리 시스템 구현 계획

## 프로젝트 개요

### 목표
조직 구성원의 업무(Task)를 체계적으로 관리하고 추적할 수 있는 시스템 구축

### 핵심 기능
1. **업무 관리** - 업무 생성, 수정, 삭제, 상태 관리
2. **스케줄링** - 업무 일정 계획 및 마감일 관리
3. **진행률 추적** - 업무별 진행 상황 시각화
4. **통계 및 리포팅** - 개인/팀/조직 단위 업무 통계
5. **권한 기반 접근 제어** - 기존 권한 시스템과 통합

## 1. 데이터베이스 설계

### 1.1 업무 공간 (Workspace) 테이블

```sql
CREATE TABLE workspace (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    organization_id BIGINT NOT NULL,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    workspace_type ENUM('PERSONAL', 'TEAM', 'PROJECT') NOT NULL DEFAULT 'TEAM',
    owner_id BIGINT NOT NULL,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP NULL,

    FOREIGN KEY (organization_id) REFERENCES organization(id) ON DELETE CASCADE,
    FOREIGN KEY (owner_id) REFERENCES user(id) ON DELETE RESTRICT,
    INDEX idx_organization_id (organization_id),
    INDEX idx_owner_id (owner_id),
    INDEX idx_workspace_type (workspace_type)
);
```

**설명**:
- `workspace_type`: PERSONAL (개인), TEAM (팀), PROJECT (프로젝트)
- `owner_id`: 워크스페이스 책임자
- Soft delete 지원 (`deleted_at`)

### 1.2 업무 (Task) 테이블

```sql
CREATE TABLE task (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    workspace_id BIGINT NOT NULL,
    parent_task_id BIGINT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    task_type ENUM('DEVELOPMENT', 'STUDY', 'MEETING', 'DOCUMENTATION', 'BUG_FIX', 'FEATURE', 'OTHER') NOT NULL,
    priority ENUM('LOW', 'MEDIUM', 'HIGH', 'URGENT') NOT NULL DEFAULT 'MEDIUM',
    status ENUM('TODO', 'IN_PROGRESS', 'IN_REVIEW', 'BLOCKED', 'COMPLETED', 'CANCELLED') NOT NULL DEFAULT 'TODO',
    progress INT DEFAULT 0 CHECK (progress >= 0 AND progress <= 100),

    assignee_id BIGINT NULL,
    reporter_id BIGINT NOT NULL,

    start_date DATE NULL,
    due_date DATE NULL,
    completed_at TIMESTAMP NULL,

    estimated_hours DECIMAL(6,2) NULL,
    actual_hours DECIMAL(6,2) NULL,

    tags JSON NULL,

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP NULL,

    FOREIGN KEY (workspace_id) REFERENCES workspace(id) ON DELETE CASCADE,
    FOREIGN KEY (parent_task_id) REFERENCES task(id) ON DELETE SET NULL,
    FOREIGN KEY (assignee_id) REFERENCES user(id) ON DELETE SET NULL,
    FOREIGN KEY (reporter_id) REFERENCES user(id) ON DELETE RESTRICT,

    INDEX idx_workspace_id (workspace_id),
    INDEX idx_assignee_id (assignee_id),
    INDEX idx_status (status),
    INDEX idx_priority (priority),
    INDEX idx_due_date (due_date),
    INDEX idx_parent_task_id (parent_task_id)
);
```

**주요 필드 설명**:
- `parent_task_id`: 상위 업무 (하위 업무 계층 구조 지원)
- `task_type`: 업무 유형 (개발, 학습, 회의, 문서화, 버그 수정 등)
- `priority`: 우선순위 (낮음, 보통, 높음, 긴급)
- `status`: 상태 (할 일, 진행 중, 리뷰 중, 차단됨, 완료, 취소)
- `progress`: 진행률 (0-100%)
- `tags`: JSON 배열로 태그 저장
- `estimated_hours` / `actual_hours`: 예상 시간 / 실제 소요 시간
