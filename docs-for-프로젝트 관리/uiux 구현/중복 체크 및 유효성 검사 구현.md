# 중복 체크 및 유효성 검사 구현

## 개요

회원가입 폼에 TanStack Form + Zod를 사용한 유효성 검사와 이메일 중복 체크 기능을 구현했습니다.

## 기술 스택

### 프론트엔드
- **TanStack Form** (`@tanstack/react-form`): 타입 안전한 폼 관리
- **Zod** (`zod@^3`): 스키마 기반 유효성 검사
- **TanStack Zod Adapter** (`@tanstack/zod-form-adapter`): Form과 Zod 연동

### 백엔드
- **Spring Boot 3**: RESTful API
- **MyBatis**: 데이터베이스 연동

## 구현 내용

### 1. 백엔드 API

#### 1.1 이메일 중복 체크 엔드포인트

**`AuthController.java`**
```java
@Operation(summary = "이메일 중복 체크", description = "회원가입 시 이메일 중복 여부를 확인합니다.")
@GetMapping("/check-email")
public ResponseEntity<Boolean> checkEmailDuplicate(@RequestParam String email) {
    boolean isDuplicate = authService.isEmailDuplicate(email);
    return ResponseEntity.ok(isDuplicate);
}
```

**`AuthService.java`**
```java
@Transactional(readOnly = true)
public boolean isEmailDuplicate(String email) {
    return userRepository.existsByEmail(email);
}
```

**API 정보**
- **Method**: `GET`
- **URL**: `/api/auth/check-email?email={email}`
- **Response**: `true` (중복) / `false` (사용 가능)

### 2. 프론트엔드 유효성 검사 스키마

**`src/shared/lib/validation/auth.schema.ts`**

```typescript
import { z } from 'zod'

export const signupSchema = z.object({
  email: z
    .string()
    .min(1, '이메일을 입력하세요')
    .email('유효한 이메일 형식이 아닙니다'),

  username: z
    .string()
    .min(2, '이름은 최소 2자 이상이어야 합니다')
    .max(50, '이름은 최대 50자까지 입력 가능합니다'),

  password: z
    .string()
    .min(8, '비밀번호는 최소 8자 이상이어야 합니다')
    .regex(/[A-Z]/, '비밀번호에 대문자가 포함되어야 합니다')
    .regex(/[a-z]/, '비밀번호에 소문자가 포함되어야 합니다')
    .regex(/[0-9]/, '비밀번호에 숫자가 포함되어야 합니다')
    .regex(/[!@#$%^&*(),.?":{}|<>]/, '비밀번호에 특수문자가 포함되어야 합니다'),
})

export type SignupFormData = z.infer<typeof signupSchema>
```

### 3. 회원가입 폼 구현

**`SignupDialog.tsx`**

#### 3.1 TanStack Form 초기화

```typescript
const form = useForm<SignupFormData>({
  defaultValues: {
    email: '',
    username: '',
    password: '',
  },
  validatorAdapter: zodValidator(),
  validators: {
    onChange: signupSchema, // 실시간 검증
  },
  onSubmit: async ({ value }) => {
    await authApi.signup(value)
    // 성공 처리...
  },
})
```

#### 3.2 이메일 중복 체크 기능

```typescript
const [emailCheckStatus, setEmailCheckStatus] = useState<
  'idle' | 'checking' | 'available' | 'duplicate'
>('idle')

const handleEmailCheck = async () => {
  const email = form.getFieldValue('email')
  if (!email || form.getFieldMeta('email')?.errors.length > 0) {
    return
  }

  setEmailCheckStatus('checking')
  try {
    const isDuplicate = await authApi.checkEmailDuplicate(email)
    setEmailCheckStatus(isDuplicate ? 'duplicate' : 'available')
  } catch (err) {
    setEmailCheckStatus('idle')
  }
}
```

#### 3.3 폼 필드 구현 예시

```typescript
<form.Field name="email">
  {(field) => (
    <div className="space-y-2">
      <Label htmlFor="signup-email">이메일</Label>
      <div className="flex gap-2">
        <div className="flex-1">
          <Input
            id="signup-email"
            type="email"
            value={field.state.value}
            onChange={(e) => {
              field.handleChange(e.target.value)
              setEmailCheckStatus('idle') // 이메일 변경 시 체크 상태 초기화
            }}
            onBlur={field.handleBlur}
          />
        </div>
        <Button
          type="button"
          variant="outline"
          onClick={handleEmailCheck}
          disabled={
            emailCheckStatus === 'checking' ||
            !field.state.value ||
            field.state.meta.errors.length > 0
          }
        >
          {emailCheckStatus === 'checking' ? '확인 중...' : '중복 확인'}
        </Button>
      </div>

      {/* 에러 메시지 */}
      {field.state.meta.errors.length > 0 && (
        <p className="text-sm text-destructive">
          {field.state.meta.errors[0]}
        </p>
      )}

      {/* 사용 가능 메시지 */}
      {emailCheckStatus === 'available' && (
        <p className="text-sm text-green-600 flex items-center gap-1">
          <Check className="h-4 w-4" />
          사용 가능한 이메일입니다
        </p>
      )}

      {/* 중복 메시지 */}
      {emailCheckStatus === 'duplicate' && (
        <p className="text-sm text-destructive flex items-center gap-1">
          <X className="h-4 w-4" />
          이미 사용 중인 이메일입니다
        </p>
      )}
    </div>
  )}
</form.Field>
```

#### 3.4 제출 버튼 제어

```typescript
<form.Subscribe selector={(state) => [state.canSubmit, state.isSubmitting]}>
  {([canSubmit, isSubmitting]) => (
    <Button
      type="submit"
      disabled={
        !canSubmit ||
        isSubmitting ||
        emailCheckStatus !== 'available' // 이메일 중복 체크 필수
      }
      className="w-full"
    >
      {isSubmitting ? '가입 중...' : '회원가입'}
    </Button>
  )}
</form.Subscribe>
```

## 주요 기능

### 1. 실시간 유효성 검사
- 사용자가 입력하는 동안 실시간으로 유효성 검사 수행
- Zod 스키마에 정의된 규칙에 따라 에러 메시지 표시

### 2. 이메일 중복 체크
- "중복 확인" 버튼 클릭 시 API 호출
- 4가지 상태 관리: `idle`, `checking`, `available`, `duplicate`
- 이메일이 변경되면 체크 상태 자동 초기화

### 3. 제출 조건
회원가입 버튼은 다음 조건을 모두 만족해야 활성화됩니다:
- ✅ 모든 필드의 유효성 검사 통과
- ✅ 이메일 중복 체크 완료 (`available` 상태)
- ✅ 제출 중이 아님

### 4. 비밀번호 강도 검증
다음 규칙을 모두 만족해야 합니다:
- 최소 8자 이상
- 대문자 포함
- 소문자 포함
- 숫자 포함
- 특수문자 포함

## 장점

### TanStack Form + Zod 조합의 장점

1. **타입 안전성**
   - Zod 스키마에서 TypeScript 타입 자동 추론
   - 런타임과 컴파일타임 모두에서 타입 검증

2. **재사용성**
   - 검증 스키마를 여러 폼에서 재사용 가능
   - 프론트엔드와 백엔드에서 동일한 검증 로직 공유 가능

3. **선언적 검증**
   - 검증 규칙을 스키마로 명시적으로 정의
   - 코드 가독성 향상

4. **세밀한 제어**
   - 필드별, 폼 전체 단위로 검증 제어 가능
   - 에러 메시지 커스터마이징 쉬움

5. **성능**
   - TanStack Form은 불필요한 리렌더링 최소화
   - 필드 단위 구독으로 최적화

## 사용 방법

### 회원가입 플로우

1. 사용자가 이메일 입력
2. "중복 확인" 버튼 클릭
3. API로 중복 여부 확인
4. 사용 가능한 이메일이면 녹색 체크 표시
5. 사용자 이름과 비밀번호 입력
6. 모든 검증 통과 시 "회원가입" 버튼 활성화
7. 제출

### 에러 처리

- **네트워크 에러**: 중복 체크 실패 시 `idle` 상태로 복귀
- **유효성 검사 실패**: 각 필드 아래에 에러 메시지 표시
- **회원가입 실패**: 폼 하단에 에러 메시지 표시

## 향후 개선 사항

1. **디바운싱 추가**
   - 이메일 입력 시 자동으로 중복 체크 (타이핑 멈춘 후 500ms)

2. **비밀번호 강도 표시**
   - 프로그레스 바로 비밀번호 강도 시각화

3. **비밀번호 확인 필드**
   - 비밀번호 재입력 필드 추가 및 일치 검증

4. **캐싱**
   - 이미 체크한 이메일은 캐싱하여 불필요한 API 호출 방지
