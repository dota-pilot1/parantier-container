# Palantier - 채용 및 업무 관리 지식 공유 사이트 구현 계획

## 프로젝트 목표

1. **DB 설계 능력 향상**: 복잡한 도메인 모델링 및 테이블 설계 경험
2. **UI 라이브러리 활용**: shadcn/ui, TailwindCSS 등 최신 UI 라이브러리 사용
3. **일정 관리**: 캘린더, 간트차트 등 스케줄링 기능 구현
4. **지식 공유**: 위키, 문서 관리 시스템 구축
5. **외부 시스템 연동**: 결제, 이메일, S3 등 외부 API 통합
6. **배치 처리**: Spring Batch를 활용한 대용량 데이터 처리
7. **데이터 정합성**: 트랜잭션, 동시성 제어 등
8. **Spring AI**: ChatGPT 연동, 문서 요약, 추천 시스템 등

## 기술 스택

### Backend
- **Framework**: Spring Boot 3.x
- **Security**: Spring Security (JWT)
- **Database**: PostgreSQL
- **ORM**: MyBatis
- **Batch**: Spring Batch
- **AI**: Spring AI (OpenAI ChatGPT)
- **Cache**: Redis
- **Storage**: AWS S3
- **Messaging**: Spring WebSocket / SSE
- **Email**: Spring Mail (SMTP)
- **Payment**: 외부 결제 API (Toss Payments, Stripe 등)

### Frontend
- **Framework**: React 18 + TypeScript
- **Build Tool**: Vite
- **Routing**: React Router DOM
- **State Management**: Zustand
- **HTTP Client**: Axios
- **UI Library**: shadcn/ui, TailwindCSS
- **Data Fetching**: React Query
- **Forms**: React Hook Form + Zod
- **Editor**: Lexical (문서 편집)
- **Calendar**: FullCalendar
- **Charts**: Recharts

### DevOps
- **Containerization**: Docker
- **CI/CD**: GitHub Actions
- **Monitoring**: Spring Actuator + Prometheus + Grafana

## DB 설계 예시

### 1. 프로젝트 관리
```sql
-- 프로젝트
CREATE TABLE projects (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    status VARCHAR(20) NOT NULL, -- PLANNING, IN_PROGRESS, COMPLETED, ON_HOLD
    start_date DATE,
    end_date DATE,
    created_by BIGINT NOT NULL REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 업무
CREATE TABLE tasks (
    id BIGSERIAL PRIMARY KEY,
    project_id BIGINT NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    status VARCHAR(20) NOT NULL, -- TODO, IN_PROGRESS, REVIEW, DONE
    priority VARCHAR(20), -- LOW, MEDIUM, HIGH, URGENT
    assigned_to BIGINT REFERENCES users(id),
    start_date DATE,
    due_date DATE,
    estimated_hours DECIMAL(5,2),
    actual_hours DECIMAL(5,2),
    parent_task_id BIGINT REFERENCES tasks(id),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 업무 댓글
CREATE TABLE task_comments (
    id BIGSERIAL PRIMARY KEY,
    task_id BIGINT NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
    user_id BIGINT NOT NULL REFERENCES users(id),
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 2. 지식 공유
```sql
-- 문서
CREATE TABLE documents (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(300) NOT NULL,
    content TEXT NOT NULL,
    category_id BIGINT REFERENCES categories(id),
    author_id BIGINT NOT NULL REFERENCES users(id),
    version INT DEFAULT 1,
    is_published BOOLEAN DEFAULT false,
    view_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 문서 버전 히스토리
CREATE TABLE document_versions (
    id BIGSERIAL PRIMARY KEY,
    document_id BIGINT NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    version INT NOT NULL,
    content TEXT NOT NULL,
    changed_by BIGINT NOT NULL REFERENCES users(id),
    change_summary VARCHAR(500),
    created_at TIMESTAMP DEFAULT NOW()
);

-- 카테고리
CREATE TABLE categories (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    parent_id BIGINT REFERENCES categories(id),
    display_order INT DEFAULT 0
);

-- 태그
CREATE TABLE tags (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL UNIQUE
);

-- 문서-태그 매핑
CREATE TABLE document_tags (
    document_id BIGINT REFERENCES documents(id) ON DELETE CASCADE,
    tag_id BIGINT REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (document_id, tag_id)
);
```

### 3. 쇼핑몰
```sql
-- 상품
CREATE TABLE products (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    price DECIMAL(10,2) NOT NULL,
    stock_quantity INT NOT NULL DEFAULT 0,
    category VARCHAR(100),
    image_url VARCHAR(500),
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 주문
CREATE TABLE orders (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL REFERENCES users(id),
    total_amount DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) NOT NULL, -- PENDING, PAID, SHIPPED, DELIVERED, CANCELLED
    payment_method VARCHAR(50),
    payment_id VARCHAR(100), -- 외부 결제 시스템 ID
    shipping_address TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 주문 항목
CREATE TABLE order_items (
    id BIGSERIAL PRIMARY KEY,
    order_id BIGINT NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
    product_id BIGINT NOT NULL REFERENCES products(id),
    quantity INT NOT NULL,
    price DECIMAL(10,2) NOT NULL, -- 주문 시점 가격
    created_at TIMESTAMP DEFAULT NOW()
);

-- 재고 변동 이력
CREATE TABLE stock_movements (
    id BIGSERIAL PRIMARY KEY,
    product_id BIGINT NOT NULL REFERENCES products(id),
    quantity_change INT NOT NULL, -- +/- 값
    movement_type VARCHAR(20) NOT NULL, -- PURCHASE, SALE, ADJUSTMENT, RETURN
    reference_id BIGINT, -- order_id 등
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 4. 일정 관리
```sql
-- 일정
CREATE TABLE schedules (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    start_datetime TIMESTAMP NOT NULL,
    end_datetime TIMESTAMP NOT NULL,
    all_day BOOLEAN DEFAULT false,
    location VARCHAR(200),
    created_by BIGINT NOT NULL REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 일정 참석자
CREATE TABLE schedule_attendees (
    schedule_id BIGINT REFERENCES schedules(id) ON DELETE CASCADE,
    user_id BIGINT REFERENCES users(id),
    status VARCHAR(20) DEFAULT 'PENDING', -- PENDING, ACCEPTED, DECLINED
    PRIMARY KEY (schedule_id, user_id)
);
```
