# 역할 기반 인가(Authorization) 기능 구현

## 개요

현재 JWT 기반 인증은 완료되었으나, 역할(Role)에 따른 실제 접근 제어가 구현되지 않은 상태입니다.
사용자 역할에 따라 API 엔드포인트와 프론트엔드 페이지 접근을 제한하는 기능을 구현합니다.

## 현재 상태

### 이미 구현된 것
- ✅ JWT 토큰 기반 인증
- ✅ User 엔티티에 `role` 필드 (UserRole enum)
- ✅ 로그인 시 역할 정보 포함

### 구현되지 않은 것
- ❌ 역할별 API 엔드포인트 접근 제어
- ❌ 역할별 프론트엔드 페이지 라우팅 제어
- ❌ 역할별 메뉴 표시/숨김
- ❌ 관리자 페이지 구현

## 구현 절차

### Phase 1: 백엔드 역할 기반 접근 제어

#### 1-1. SecurityConfig 역할별 엔드포인트 설정
**파일**: `parantier-api/src/main/java/com/mapo/palantier/config/SecurityConfig.java`

**작업 내용:**
```java
.authorizeHttpRequests(auth -> auth
    // 공개 엔드포인트
    .requestMatchers("/api/auth/signup", "/api/auth/login", "/api/auth/check-email").permitAll()
    .requestMatchers("/actuator/health", "/actuator/health/**").permitAll()
    .requestMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll()

    // 관리자 전용 엔드포인트
    .requestMatchers("/api/admin/**").hasRole("ADMIN")

    // 일반 사용자 + 관리자 접근 가능
    .requestMatchers("/api/user/**").hasAnyRole("USER", "ADMIN")

    // 나머지는 인증만 필요
    .requestMatchers("/api/**").authenticated()
    .anyRequest().authenticated()
)
```

**검증 방법:**
- Swagger UI에서 ADMIN 전용 엔드포인트 테스트
- USER 토큰으로 접근 시 403 에러 확인

#### 1-2. 컨트롤러 메서드 레벨 권한 설정
**파일**: 각 Controller 파일

**작업 내용:**
```java
// 예시: UserController.java
@RestController
@RequestMapping("/api/admin/users")
public class AdminUserController {

    // 메서드 레벨에서도 권한 체크 가능
    @PreAuthorize("hasRole('ADMIN')")
    @GetMapping
    public ResponseEntity<List<User>> getAllUsers() {
        // ...
    }

    @PreAuthorize("hasRole('ADMIN')")
    @PutMapping("/{userId}/role")
    public ResponseEntity<User> updateUserRole(
        @PathVariable Long userId,
        @RequestBody UpdateRoleRequest request
    ) {
        // ...
    }
}
```

**필요한 설정:**
```java

    // ...
}
```

### Phase 2: 프론트엔드 라우트 가드 구현

#### 2-1. ProtectedRoute 컴포넌트 생성
**파일**: `parantier-front/src/shared/ui/ProtectedRoute.tsx`

**작업 내용:**
```typescript
import { ReactNode } from 'react'
import { Navigate } from '@tanstack/react-router'
import { useStore } from '@tanstack/react-store'
import { authStore } from '@/entities/user/model/authStore'

interface ProtectedRouteProps {
  children: ReactNode
  allowedRoles?: string[]
}

export function ProtectedRoute({ children, allowedRoles }: ProtectedRouteProps) {
  const auth = useStore(authStore, (state) => state)

  // 1. 로그인 체크
  if (!auth.isAuthenticated) {
    return <Navigate to="/login" />
  }

  // 2. 역할 체크
  if (allowedRoles && auth.user && !allowedRoles.includes(auth.user.role)) {
    return <Navigate to="/403" />
  }

  return <>{children}</>
}
```

#### 3-2. 메뉴 필터링 Hook 생성
**파일**: `parantier-front/src/shared/hooks/useFilteredMenu.ts`

**작업 내용:**
```typescript
import { useMemo } from 'react'
import { useStore } from '@tanstack/react-store'
import { authStore } from '@/entities/user/model/authStore'
import { menuConfig, MenuItem } from '@/shared/config/menuConfig'

export function useFilteredMenu() {
  const auth = useStore(authStore, (state) => state)

  const filteredMenu = useMemo(() => {
    if (!auth.user) return []

    const filterByRole = (items: MenuItem[]): MenuItem[] => {
      return items
        .filter(item => item.allowedRoles.includes(auth.user!.role))
        .map(item => ({
          ...item,
          children: item.children ? filterByRole(item.children) : undefined,
        }))
    }

    return filterByRole(menuConfig)
  }, [auth.user])

  return filteredMenu
}
```

```typescript
export const adminUserApi = {
  // 전체 사용자 조회
  getAllUsers: async (params?: {
    page?: number
    size?: number
    search?: string
  }) => {
    const response = await apiClient.get('/admin/users', { params })
    return response.data
  },

  // 역할 변경
  updateUserRole: async (userId: number, role: string) => {
    const response = await apiClient.put(`/admin/users/${userId}/role`, { role })
    return response.data
  },

  // 계정 상태 변경
  updateUserStatus: async (userId: number, isActive: boolean) => {
    const response = await apiClient.put(`/admin/users/${userId}/status`, { isActive })
    return response.data
  },
}
```

## 구현 순서 요약

```
1. 백엔드 SecurityConfig 수정
   └─> API 엔드포인트별 역할 제한 설정

2. 백엔드 관리자 API 구현
   └─> AdminUserController, AdminUserService 생성

3. 프론트엔드 ProtectedRoute 컴포넌트
   └─> 역할 기반 페이지 접근 제어

4. 403 Forbidden 페이지
   └─> 권한 없을 때 표시할 페이지

5. 메뉴 설정 및 필터링
   └─> menuConfig, useFilteredMenu Hook

6. Header 컴포넌트 수정
   └─> 역할별 메뉴 표시/숨김

7. 관리자 페이지 구현
   └─> AdminLayout, AdminUsersPage

8. API 에러 처리
   └─> 403 에러 핸들링
```

## 테스트 체크리스트

### 백엔드 테스트
- [ ] USER 역할로 `/api/admin/**` 접근 시 403 에러
- [ ] ADMIN 역할로 `/api/admin/**` 접근 시 정상 응답
- [ ] 인증 없이 공개 API 접근 가능
- [ ] Swagger UI에서 역할별 API 테스트

### 프론트엔드 테스트
- [ ] USER로 로그인 시 관리자 메뉴 안 보임
- [ ] ADMIN으로 로그인 시 관리자 메뉴 보임
- [ ] USER가 `/admin` 접근 시 403 페이지로 리다이렉트
- [ ] ADMIN이 `/admin` 접근 시 정상 표시
- [ ] 비로그인 상태에서 보호된 페이지 접근 시 로그인 페이지로 이동
