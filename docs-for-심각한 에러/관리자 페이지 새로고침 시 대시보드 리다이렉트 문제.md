# 관리자 페이지 새로고침 시 대시보드 리다이렉트 문제

## 문제 상황

### 증상
- `/admin/organizations` 등 관리자 페이지에서 새로고침(F5)을 하면 `/dashboard`로 리다이렉트됨
- 사용자가 현재 페이지를 유지하지 못하고 대시보드로 이동함
- 모든 `/admin/*` 경로에서 동일한 문제 발생

### 발생 일시
2026-03-25

### 영향 범위
- `/admin/users` - 사용자 관리
- `/admin/organizations` - 조직 관리
- `/admin/menus` - 메뉴 관리
- `/admin/workspace` - 업무 관리
- `/admin/authorities` - 권한 관리
- `/admin/roles` - 역할 관리

## 원인 분석

### 근본 원인: Race Condition
페이지 새로고침 시 인증 상태 복원과 라우트 가드 실행 간에 타이밍 문제 발생

### 상세 분석

#### 1. 인증 상태 복원 프로세스
```typescript
// App.tsx
function App() {
  useEffect(() => {
    authActions.restoreAuth()  // 비동기 API 호출
  }, [])

  return <RouterProvider router={router} />
}
```

```typescript
// authStore.ts
restoreAuth: async () => {
  const accessToken = localStorage.getItem('accessToken')
  const refreshToken = localStorage.getItem('refreshToken')

  if (!accessToken || !refreshToken) {
    return
  }

  try {
    const user = await authApi.getCurrentUser()  // 🔴 비동기 API 호출
    const roles = getRolesFromToken(accessToken)
    const authorities = getAuthoritiesFromToken(accessToken)

    authStore.setState((state) => ({
      ...state,
      user: { ...user, roles, authorities },
      accessToken,
      refreshToken,
      isAuthenticated: true,
    }))
  } catch (error) {
    authActions.logout()
  }
}
```

#### 2. 라우트 가드 실행 (문제의 코드)
```typescript
// routes/index.tsx - 수정 전
const requireRole = (requiredRole: string) => {
  const auth = authStore.state

  if (!auth.isAuthenticated) {
    toast.error('로그인이 필요합니다')
    throw redirect({ to: '/dashboard' })
  }

  // 🔴 문제: user 객체가 null인 상태에서 체크
  if (auth.user?.role === requiredRole) {
    return
  }

  // 권한이 없으면 접근 차단
  toast.error('접근 권한이 없습니다', {
    description: `관리자만 접근할 수 있습니다.`,
  })
  throw redirect({ to: '/dashboard' })
}
```

#### 3. 문제 발생 시나리오
```
1. 페이지 새로고침 (F5)
   ↓
2. App 컴포넌트 마운트
   ↓
3. useEffect에서 restoreAuth() 호출 (비동기)
   ↓
4. RouterProvider 렌더링 시작
   ↓
5. beforeLoad에서 requireRole() 실행 (동기)  ⚠️
   ↓
6. authStore.state.user가 아직 null  ❌
   ↓
7. 권한 체크 실패로 판단
   ↓
8. /dashboard로 리다이렉트  ❌
   ↓
9. (백그라운드) restoreAuth() 완료  🕐 (너무 늦음)
```

#### 4. 초기 상태 문제
```typescript
// authStore.ts
const initialState: AuthState = {
  user: null,  // 🔴 초기값이 null
  accessToken,
  refreshToken,
  isAuthenticated: !!(accessToken && refreshToken),
  // ⚠️ 토큰은 있지만 user는 null
}
```

**핵심 문제점:**
- `isAuthenticated: true` (토큰이 localStorage에 있음)
- `user: null` (아직 API 호출 전)
- 이 상태에서 `auth.user?.role`을 체크하면 `undefined` 반환
- 권한 체크 실패로 리다이렉트 발생

## 해결 방법

### 수정 코드
```typescript
// routes/index.tsx - 수정 후
const requireRole = async (requiredRole: string) => {
  const auth = authStore.state

  if (!auth.isAuthenticated) {
    toast.error('로그인이 필요합니다')
    throw redirect({ to: '/dashboard' })
  }

  // ✅ 인증 상태가 복원될 때까지 기다림
  if (!auth.user) {
    // 짧은 대기 후 재확인 (최대 3초)
    for (let i = 0; i < 30; i++) {
      await new Promise((resolve) => setTimeout(resolve, 100))
      const currentAuth = authStore.state
      if (currentAuth.user) {
        break
      }
    }
  }

  // ✅ 다시 확인
  const finalAuth = authStore.state

  // User role 체크
  if (finalAuth.user?.role === requiredRole) {
    return
  }

  // 권한이 없으면 접근 차단
  toast.error('접근 권한이 없습니다', {
    description: `관리자만 접근할 수 있습니다.`,
  })
  throw redirect({ to: '/dashboard' })
}
```

### 해결 전략

#### 1. 비동기 함수로 변경
- `const requireRole = (requiredRole: string) => {`
- → `const requireRole = async (requiredRole: string) => {`

#### 2. 대기 로직 추가
```typescript
if (!auth.user) {
  // 최대 3초간 100ms 간격으로 30번 체크
  for (let i = 0; i < 30; i++) {
    await new Promise((resolve) => setTimeout(resolve, 100))
    const currentAuth = authStore.state
    if (currentAuth.user) {
      break  // user 복원 완료
    }
  }
}
```

#### 3. 복원 완료 후 재확인
```typescript
const finalAuth = authStore.state
if (finalAuth.user?.role === requiredRole) {
  return  // 권한 있음
}
```

## 학습 포인트

### 1. Race Condition 패턴
- 비동기 작업(API 호출)과 동기 작업(라우트 가드)의 실행 순서 불일치
- React의 useEffect와 라우터의 beforeLoad 간 타이밍 이슈

### 2. 해결 패턴: Polling
```typescript
for (let i = 0; i < maxAttempts; i++) {
  await new Promise(resolve => setTimeout(resolve, interval))
  if (condition) break
}
```
- 최대 시도 횟수 제한 (30회)
- 짧은 간격 (100ms)
- 조건 만족 시 즉시 탈출

### 3. 대안 고려사항

#### 옵션 A: 현재 방식 (Polling)
- ✅ 장점: 구현 간단
- ❌ 단점: 3초 최대 대기 시간

#### 옵션 B: Promise/Observable 기반
```typescript
const waitForUser = () => new Promise((resolve) => {
  const unsubscribe = authStore.subscribe(() => {
    if (authStore.state.user) {
      unsubscribe()
      resolve(authStore.state.user)
    }
  })
})
```
- ✅ 장점: 이벤트 기반, 더 정확
- ❌ 단점: 구현 복잡도 증가

#### 옵션 C: Suspense 패턴 (React 18+)
```typescript
const userResource = createResource(authActions.restoreAuth)
```
- ✅ 장점: React 표준 패턴
- ❌ 단점: TanStack Router와의 통합 복잡

fix: 조직 관리 페이지 새로고침 시 대시보드로 리다이렉트되는 문제 수정

문제:
- /admin/organizations 페이지에서 새로고침(F5)하면 대시보드로 리다이렉트됨
- beforeLoad의 requireRole 함수가 동기적으로 실행되어 authStore.restoreAuth() 완료 전에 체크

- requireRole 함수를 async로 변경

- requireAuthority와 동일한 대기 로직 적용
