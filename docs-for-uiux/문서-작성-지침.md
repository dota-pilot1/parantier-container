# 문서 작성 지침

## 1. 파일 이름 규칙

### ❌ 사용 금지: .txt 파일
**문제**: `.txt` 파일은 한글 인코딩 문제가 발생합니다.

```
ag-grid 기본 사용법.txt  ❌ (한글 깨짐 발생)
```

### ✅ 권장: .md 파일 (마크다운)
**이유**: 마크다운 파일은 기본적으로 UTF-8 인코딩이 적용되어 한글이 깨지지 않습니다.

```
ag-grid-기본-사용법.md  ✅ (한글 정상 표시)
ag-grid-체크박스.md      ✅ (한글 정상 표시)
```

---

## 2. 파일명 작성 규칙

### 공백 대신 하이픈(-) 사용
```
❌ ag grid 기본 사용법.md
✅ ag-grid-기본-사용법.md
```

### 소문자 사용 (선택사항)
```
✅ ag-grid-기본-사용법.md
✅ AG-Grid-기본-사용법.md  (둘 다 가능)
```

---

## 3. 인코딩 문제 해결

### 이미 생성된 .txt 파일이 있다면

```bash
# 1. 기존 txt 파일 삭제
rm "ag-grid 기본 사용법.txt"

# 2. 마크다운 파일로 다시 생성
# (파일명에 공백 대신 하이픈 사용)
touch ag-grid-기본-사용법.md
```

### VSCode에서 확인하는 방법
1. 파일을 열었을 때 한글이 깨져 보이면 인코딩 문제
2. 우측 하단에서 인코딩 확인 (UTF-8이어야 함)
3. `.txt` 파일은 기본 인코딩이 시스템마다 다를 수 있음

## 5. 마크다운 기본 문법

### 제목
```markdown
# H1 제목
## H2 제목
### H3 제목
```

### 코드 블록
````markdown
```typescript
const example = 'code here'
```
````

### 강조
```markdown
**굵게**
*기울임*
`코드`
```

### 리스트
```markdown
- 항목 1
- 항목 2
  - 하위 항목

1. 순서 1
2. 순서 2
```

## 6. 요약

**핵심 규칙:**
1. `.txt` 파일 사용 금지 (한글 깨짐)
2. `.md` 파일 사용 (마크다운)
3. 파일명에 공백 대신 하이픈(-) 사용
4. UTF-8 인코딩 확인

**예시:**
```
❌ ag grid 기본 사용법.txt
✅ ag-grid-기본-사용법.md
```
