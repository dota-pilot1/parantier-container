# 권한 관련 에러 디버깅

## 발생한 주요 에러들

### 1. NullPointerException: Authentication is null

#### 에러 메시지
```
java.lang.NullPointerException: Cannot invoke "org.springframework.security.core.Authentication.getAuthorities()" because "auth" is null
	at com.mapo.palantier.menu.presentation.MenuController.getMenuTree(MenuController.java:27)
```

#### 원인
- 로그인한 상태에서도 `Authentication` 객체가 null이 되는 문제
- JWT 토큰이 SecurityContext에 제대로 설정되지 않음

#### 가능한 원인들

**1) 프론트엔드에서 토큰을 보내지 않음**
- localStorage에 토큰이 저장되어 있지만, API 요청 시 Authorization 헤더에 포함되지 않음
- axios interceptor가 제대로 동작하지 않음

**2) JWT 필터가 토큰을 파싱하지 못함**
- 토큰 형식이 잘못됨 (Bearer prefix 누락 등)
- 토큰이 만료됨
- 토큰 검증 실패

**3) SecurityContext가 초기화됨**
- 요청 처리 중 SecurityContext가 비워짐
- 다른 필터에서 Authentication을 제거함

#### 해결 방법

**임시 해결책: null 체크 추가**
```java
@GetMapping("/tree")
public ResponseEntity<List<MenuResponse>> getMenuTree(Authentication auth) {
    String role = "USER"; // 기본값

    if (auth != null && auth.getAuthorities() != null) {
        role = auth.getAuthorities().stream()
                .findFirst()
                .map(Object::toString)
                .map(r -> r.replace("ROLE_", ""))
                .orElse("USER");
    }

    List<Menu> menus = menuService.getMenuTreeByRole(role);
    return ResponseEntity.ok(MenuResponse.fromList(menus));
}
```

**근본적인 해결책: 로그 추가 및 디버깅**

1. **JWT 필터에 로그 추가**
```java
@Override
protected void doFilterInternal(
        HttpServletRequest request,
        HttpServletResponse response,
        FilterChain filterChain
) throws ServletException, IOException {
    try {
        String token = getTokenFromRequest(request);

        logger.info("JWT Filter - Path: {}, Token exists: {}",
                    request.getRequestURI(), token != null);

        if (StringUtils.hasText(token) && jwtTokenProvider.validateToken(token)) {
            String email = jwtTokenProvider.getEmailFromToken(token);
            String role = jwtTokenProvider.getRoleFromToken(token);

            logger.info("JWT Filter - Email: {}, Role: {}", email, role);

            UsernamePasswordAuthenticationToken authentication =
                    new UsernamePasswordAuthenticationToken(
                            email,
                            null,
                            Collections.singletonList(new SimpleGrantedAuthority(role))
                    );

            authentication.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
            SecurityContextHolder.getContext().setAuthentication(authentication);

            logger.info("JWT Filter - Authentication set successfully");
        } else {
            logger.info("JWT Filter - No valid token found");
        }
    } catch (Exception e) {
        logger.error("Could not set user authentication in security context", e);
    }

    filterChain.doFilter(request, response);
}
```

2. **프론트엔드 토큰 전송 확인**

브라우저 개발자 도구(F12) → Network 탭에서 확인:
- Request Headers에 `Authorization: Bearer [토큰]`이 있는지
- 토큰 값이 올바른지

3. **Axios Interceptor 확인**
```typescript
// parantier-front/src/shared/api/axios.ts
apiClient.interceptors.request.use(
  (config) => {
    const state = authStore.state
    if (state.accessToken) {
      config.headers.Authorization = `Bearer ${state.accessToken}`
    }
    return config
  },
  (error) => {
    return Promise.reject(error)
  }
)
```

### 2. isAuthenticated가 false로 초기화되는 문제

#### 원인
- localStorage에 토큰이 있어도 `isAuthenticated`가 `false`로 고정됨
- 페이지 새로고침 시 인증 상태가 초기화됨

#### 해결 방법

**authStore.ts 수정**
```typescript
// 초기 상태
const accessToken = localStorage.getItem('accessToken')
const refreshToken = localStorage.getItem('refreshToken')

const initialState: AuthState = {
  user: null,
  accessToken,
  refreshToken,
  isAuthenticated: !!(accessToken && refreshToken), // 토큰이 둘 다 있으면 인증된 것으로 간주
}
```

**기존 코드 (잘못된 방식)**
```typescript
const initialState: AuthState = {
  user: null,
  accessToken: localStorage.getItem('accessToken'),
  refreshToken: localStorage.getItem('refreshToken'),
  isAuthenticated: false, // ❌ 항상 false로 고정
}
```

### 3. 메뉴가 조회되지 않는 문제

#### 원인
`useMenuTree` 훅에서 `enabled: auth.isAuthenticated` 설정으로 인해 인증되지 않으면 메뉴를 조회하지 않음

#### 해결 방법

**방법 1: enabled 조건 제거 (비인증 사용자도 메뉴 조회 가능)**
```typescript
export function useMenuTree() {
  const auth = useStore(authStore, (state) => state)

  return useQuery({
    queryKey: ['menus', 'tree', auth.isAuthenticated],
    queryFn: menuApi.getMenuTree,
    // enabled 제거 - 항상 메뉴 조회
    staleTime: 5 * 60 * 1000,
  })
}
```

**방법 2: SecurityConfig에서 /api/menus/** permitAll 설정**
```java
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/menus/**").permitAll() // 메뉴 조회는 인증 없이 가능
            .requestMatchers("/api/**").authenticated()
            .anyRequest().authenticated()
        );

    return http.build();
}
```

### 4. MenuType.CATEGORY enum not found 에러

#### 에러 메시지
```
java.lang.IllegalArgumentException: No enum constant com.mapo.palantier.menu.domain.MenuType.CATEGORY
```

#### 원인
- 데이터베이스에 `CATEGORY` 타입을 저장했지만, Java enum에는 정의되지 않음

#### 해결 방법

**MenuType.java에 CATEGORY 추가**
```java
public enum MenuType {
    HEADER,   // 헤더 메뉴 (직접 페이지로 이동)
    CATEGORY, // 카테고리 메뉴 (사이드바 토글, 페이지 없음)
    SIDE,     // 사이드바 메뉴
    SUB       // 서브 메뉴
}
```

### 5. Port 8080 already in use 에러

#### 원인
- 여러 개의 Gradle bootRun 프로세스가 동시에 실행됨
- 이전 프로세스가 제대로 종료되지 않음

#### 해결 방법

**모든 포트 8080 프로세스 종료**
```bash
lsof -ti:8080 | xargs kill -9
```

**모든 gradlew 프로세스 종료**
```bash
pkill -f gradlew
```

**결합 명령어**
```bash
lsof -ti:8080 | xargs kill -9 && pkill -f gradlew
```

## 디버깅 체크리스트

### 프론트엔드 체크리스트

- [ ] localStorage에 `accessToken`과 `refreshToken`이 저장되어 있는가?
- [ ] authStore의 `isAuthenticated`가 `true`인가?
- [ ] axios interceptor가 Authorization 헤더를 추가하는가?
- [ ] Network 탭에서 Authorization 헤더가 전송되는가?
- [ ] 토큰 값이 "Bearer " prefix와 함께 전송되는가?

### 백엔드 체크리스트

- [ ] JWT 필터가 실행되는가? (로그 확인)
- [ ] 토큰이 올바르게 파싱되는가? (로그 확인)
- [ ] SecurityContext에 Authentication이 설정되는가? (로그 확인)
- [ ] Controller에서 Authentication이 null이 아닌가?
- [ ] SecurityConfig에서 해당 endpoint가 인증을 요구하는가?

## 디버깅 로그 확인 방법

### 백엔드 로그 확인

```bash
# Gradle 실행 로그 확인
cd parantier-api && ./gradlew bootRun
```

**확인해야 할 로그:**
```
JWT Filter - Path: /api/menus/tree, Token exists: true
JWT Filter - Email: user@example.com, Role: ROLE_ADMIN
JWT Filter - Authentication set successfully
```

**문제가 있는 경우:**
```
JWT Filter - Path: /api/menus/tree, Token exists: false
JWT Filter - No valid token found
```

### 프론트엔드 디버깅

**브라우저 Console에서 확인:**
```javascript
// localStorage 확인
console.log(localStorage.getItem('accessToken'))
console.log(localStorage.getItem('refreshToken'))

// authStore 상태 확인 (React DevTools 필요)
```

**Network 탭에서 확인:**
1. F12 → Network 탭 열기
2. `/api/menus/tree` 요청 클릭
3. Headers 탭에서 Request Headers 확인
4. `Authorization: Bearer [토큰]`이 있는지 확인
