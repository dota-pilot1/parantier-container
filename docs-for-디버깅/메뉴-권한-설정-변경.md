# 메뉴 권한 설정 변경

## 문제 상황
- 대시보드 메뉴의 `required_role`이 `'USER'`로 설정되어 있음
- 현재 쿼리 로직: `WHERE required_role IS NULL OR required_role = #{role}`
- ADMIN 사용자가 로그인하면 `required_role = 'ADMIN'`인 메뉴만 표시됨
- 결과: ADMIN 사용자가 대시보드 메뉴를 볼 수 없음

## 해결 방법

### 옵션 1: 데이터 수정 (선택한 방법)
대시보드를 모든 사용자에게 표시되도록 `required_role`을 `NULL`로 변경

```sql
UPDATE menus
SET required_role = NULL
WHERE name = '대시보드';
```

**장점**:
- 간단하고 명확한 해결책
- 대시보드는 일반적으로 모든 사용자에게 보여야 함
- SQL 로직 변경 없이 데이터만 수정

**결과**:
- 대시보드: `required_role = NULL` → 모든 사용자
- 관리자: `required_role = 'ADMIN'` → ADMIN만

### 옵션 2: SQL 로직 수정 (시도했으나 취소)
권한 계층을 고려하도록 쿼리 수정

```sql
WHERE (
    required_role IS NULL
    OR required_role = #{role}
    OR (#{role} = 'ADMIN' AND required_role = 'USER')  -- ADMIN이 USER 메뉴도 볼 수 있게
)
```

**단점**:
- 복잡한 권한 로직
- 향후 권한이 추가되면 계속 수정 필요
- 비즈니스 로직이 SQL에 하드코딩됨

## 권한 설계 원칙

### required_role 의미
- `NULL`: 모든 사용자에게 표시 (비로그인 포함)
- `'USER'`: USER 권한 필요 (로그인 필요)
- `'ADMIN'`: ADMIN 권한 필요 (관리자만)

### 메뉴별 권한 설정 가이드
- **대시보드**: `NULL` (모든 사용자)
- **내 정보**: `'USER'` (로그인한 사용자)
- **관리자 메뉴**: `'ADMIN'` (관리자만)
- **공지사항**: `NULL` (모든 사용자)
- **설정**: `'USER'` (로그인한 사용자)

## 향후 개선 방안

만약 권한 계층(USER < ADMIN)을 구현하려면:

1. **DB에 권한 계층 테이블 추가**
   ```sql
   CREATE TABLE role_hierarchy (
       parent_role VARCHAR(20),
       child_role VARCHAR(20)
   );

   INSERT INTO role_hierarchy VALUES ('ADMIN', 'USER');
   ```

2. **애플리케이션 레벨에서 처리**
   - MenuService에서 권한 계층 확인
   - Spring Security의 RoleHierarchy 활용

3. **또는 비트 플래그 사용**
   ```sql
   -- role_flags: 1=USER, 2=ADMIN, 4=SUPER_ADMIN
   WHERE (required_role_flags & #{userRoleFlags}) > 0
   ```
