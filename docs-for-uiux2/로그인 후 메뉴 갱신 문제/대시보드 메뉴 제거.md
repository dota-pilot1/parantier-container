# 대시보드 메뉴 제거

## 배경

대시보드는 애플리케이션의 홈 페이지와 동일한 역할을 하므로, 헤더에 별도의 "대시보드" 메뉴를 표시할 필요가 없습니다.

## 변경 사항

### 1. DB에서 대시보드 메뉴 삭제

```sql
DELETE FROM menus WHERE name = '대시보드';
```

**실행 결과**:
```
id |   name    | menu_type | required_role | parent_id | order_num
----+-----------+-----------+---------------+-----------+-----------
 10 | 유저 관리 | SIDE      | ADMIN         |         2 |         1
  2 | 관리자    | HEADER    | ADMIN         |           |         2
 11 | 메뉴 관리 | SIDE      | ADMIN         |         2 |         2
```

- 대시보드 메뉴(id=1) 삭제됨
- "관리자" 헤더 메뉴만 남음
- 사이드바 메뉴: "유저 관리", "메뉴 관리"

### 2. Frontend 라우팅 정리

**파일**: `parantier-front/src/App.tsx`

**변경 전**:
```tsx
const currentHeaderPath = useMemo(() => {
  if (currentPath.startsWith('/admin')) return '/admin'
  if (currentPath.startsWith('/dashboard')) return '/dashboard'  // ❌ 불필요
  return null
}, [currentPath])
```

**변경 후**:
```tsx
const currentHeaderPath = useMemo(() => {
  if (currentPath.startsWith('/admin')) return '/admin'
  return null  // ✅ dashboard는 사이드바 없음
}, [currentPath])
```

## 사용자 경험 개선

### Before (변경 전)
```
┌─────────────────────────────────────┐
│ Palantier  [대시보드] [관리자]     │  ← 헤더 메뉴
└─────────────────────────────────────┘
```

- 홈 페이지에 접속해도 "대시보드" 메뉴가 헤더에 표시됨
- 중복되고 혼란스러운 네비게이션

### After (변경 후)
```
┌─────────────────────────────────────┐
│ Palantier  [관리자]                │  ← 헤더 메뉴
└─────────────────────────────────────┘
```

- 홈 페이지(/)에 접속하면 대시보드 페이지가 표시됨
- 헤더에는 "관리자" 메뉴만 표시
- 깔끔하고 직관적인 UI

## 라우팅 동작

### 루트 경로 (/)
```
/ → /dashboard (자동 리다이렉트) → MainPage 표시
사이드바: 없음
헤더 메뉴: "관리자"만
```

### 관리자 경로 (/admin/*)
```
/admin/users → UsersPage 표시
/admin/menus → MenusPage 표시
사이드바: "유저 관리", "메뉴 관리"
헤더 메뉴: "관리자"
```

## 장점

1. **UI 단순화**: 불필요한 메뉴 항목 제거
2. **사용자 혼란 감소**: 홈 = 대시보드라는 명확한 구조
3. **일관된 UX**: 대부분의 웹 앱에서 사용하는 패턴
4. **공간 절약**: 헤더 공간을 효율적으로 사용

## 로고 클릭 네비게이션 추가

대시보드 메뉴가 헤더에서 제거되었으므로, 로고 클릭 시 홈으로 이동하는 기능을 추가했습니다.

**파일**: `parantier-front/src/widgets/header/Header.tsx`

```tsx
const handleLogoClick = (e: React.MouseEvent<HTMLAnchorElement>) => {
  e.preventDefault()
  window.history.pushState({}, '', '/dashboard')
  window.dispatchEvent(new Event('navigate'))
}

return (
  <header className="border-b border-border bg-card">
    <div className="max-w-full px-6 py-3">
      <div className="flex items-center justify-between">
        <div className="flex items-center gap-8">
          <a
            href="/dashboard"
            onClick={handleLogoClick}
            className="text-xl font-bold text-primary hover:opacity-80 transition-opacity cursor-pointer"
          >
            Palantier
          </a>
          {/* ... */}
        </div>
      </div>
    </div>
  </header>
)
```

**변경 사항**:
- `<h1>` 태그를 `<a>` 태그로 변경
- `handleLogoClick` 핸들러 추가
- hover 효과 추가 (`hover:opacity-80`)
- cursor 스타일 명시 (`cursor-pointer`)

**UX 개선**:
- ✅ 웹의 표준 패턴 준수 (로고 = 홈)
- ✅ 사용자가 직관적으로 홈으로 돌아갈 수 있음
- ✅ 별도의 "대시보드" 메뉴 없이도 접근 가능

## 참고 사항

- 대시보드 **페이지**는 여전히 존재함 (MainPage 컴포넌트)
- `/dashboard` 경로로도 접근 가능
- 헤더 **메뉴**에서만 제거됨
- **로고 클릭 시 `/dashboard`로 이동** (표준 웹 패턴)
