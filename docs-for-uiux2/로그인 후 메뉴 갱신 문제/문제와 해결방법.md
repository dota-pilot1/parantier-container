# 로그인 후 헤더 메뉴가 즉시 표시되지 않는 문제

## 문제 상황

### 증상
- 사용자가 로그인 버튼을 클릭한 후 헤더에 메뉴가 바로 표시되지 않음
- 페이지를 새로고침해야만 헤더 메뉴("대시보드", "관리자")가 나타남
- 로그아웃 후에도 헤더 메뉴가 그대로 남아있음

### 발생 원인

**React Query 캐시 무효화 누락**

1. 로그인/로그아웃 시 사용자 권한이 변경됨
2. 메뉴 API는 사용자 권한에 따라 다른 메뉴를 반환함
   - 비로그인: `required_role = NULL`인 메뉴만
   - USER: NULL 또는 `required_role = 'USER'`인 메뉴
   - ADMIN: NULL 또는 `required_role = 'ADMIN'`인 메뉴
3. 그러나 React Query는 이전에 캐시된 메뉴 데이터를 계속 사용함
4. 결과: 권한이 바뀌어도 메뉴가 갱신되지 않음

### 영향 범위
- Header 컴포넌트 (헤더 메뉴)
- Sidebar 컴포넌트 (사이드바 메뉴)
- 모든 권한 기반 네비게이션

## 해결 방법

### 1. LoginForm에 Query Invalidation 추가

로그인 성공 시 메뉴 쿼리 캐시를 무효화하여 재요청을 트리거합니다.

**파일**: `parantier-front/src/features/auth/login/LoginForm.tsx`

```tsx
import { useQueryClient } from '@tanstack/react-query'

export function LoginForm() {
  const queryClient = useQueryClient()
  // ...

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault()
    setError('')
    setIsLoading(true)

    try {
      const response = await authApi.login({ email, password })

      authActions.login(response.accessToken, response.refreshToken, {
        email: response.email,
        username: response.username,
        role: response.role,
      })

      // ✅ 로그인 성공 시 메뉴 쿼리 무효화
      queryClient.invalidateQueries({ queryKey: ['menus'] })

      setEmail('')
      setPassword('')
    } catch (err: any) {
      setError(err.response?.data?.message || '로그인에 실패했습니다.')
    } finally {
      setIsLoading(false)
    }
  }
}
```

### 2. LogoutButton에 Query Invalidation 추가

로그아웃 시에도 메뉴 쿼리를 무효화하여 비로그인 사용자용 메뉴로 변경합니다.

**파일**: `parantier-front/src/features/auth/logout/LogoutButton.tsx`

```tsx
import { useQueryClient } from '@tanstack/react-query'

export function LogoutButton() {
  const queryClient = useQueryClient()
  const [isLoading, setIsLoading] = useState(false)

  const handleLogout = async () => {
    setIsLoading(true)

    try {
      await authApi.logout()
    } catch (err) {
      console.error('Logout failed:', err)
    } finally {
      authActions.logout()

      // ✅ 로그아웃 시 메뉴 쿼리 무효화
      queryClient.invalidateQueries({ queryKey: ['menus'] })

      setIsLoading(false)
    }
  }
}
```

## 작동 원리

### React Query Invalidation Flow

1. **로그인 전**:
   ```
   useMenuTree() → React Query 캐시 확인 → 비로그인 메뉴 반환
   ```

2. **로그인 시**:
   ```
   authApi.login()
   → authActions.login() (토큰 저장)
   → queryClient.invalidateQueries({ queryKey: ['menus'] })
   → React Query 캐시 무효화
   → useMenuTree() 재실행
   → menuApi.getMenuTree() 호출 (JWT 토큰 포함)
   → 서버가 ADMIN 권한 메뉴 반환
   → Header 컴포넌트 자동 재렌더링
   → 헤더에 "대시보드", "관리자" 메뉴 표시
   ```

3. **로그아웃 시**:
   ```
   authActions.logout() (토큰 삭제)
   → queryClient.invalidateQueries({ queryKey: ['menus'] })
   → React Query 캐시 무효화
   → useMenuTree() 재실행
   → menuApi.getMenuTree() 호출 (JWT 토큰 없음)
   → 서버가 비로그인 메뉴 반환
   → 헤더 메뉴 숨김 처리
   ```

### invalidateQueries의 역할

```tsx
queryClient.invalidateQueries({ queryKey: ['menus'] })
```

- `['menus']` 키로 시작하는 모든 쿼리 캐시를 무효화
- `['menus', 'tree']`, `['menus', 'list']` 등 모두 포함
- 무효화된 쿼리는 다음 렌더링 시 자동으로 재요청됨
- 사용자는 별도의 새로고침 없이 최신 데이터를 볼 수 있음

## 추가 개선 사항

### 1. 서버 측 권한 필터링 확인

**파일**: `parantier-api/src/main/resources/mybatis/mapper/MenuMapper.xml`

```xml
<select id="findMenuTreeByRole" resultMap="MenuResultMap">
    WITH RECURSIVE menu_tree AS (
        SELECT * FROM menus
        WHERE parent_id IS NULL AND is_active = true
        UNION ALL
        SELECT m.* FROM menus m
        INNER JOIN menu_tree mt ON m.parent_id = mt.id
        WHERE m.is_active = true
    )
    SELECT * FROM menu_tree
    WHERE required_role IS NULL OR required_role = #{role}
    ORDER BY parent_id NULLS FIRST, order_num
</select>
```

**권한별 필터링 결과**:
- **비로그인 (role = null)**: `required_role IS NULL`인 메뉴만
- **ADMIN (role = 'ADMIN')**: `required_role IS NULL` 또는 `required_role = 'ADMIN'`인 메뉴

### 2. 메뉴 API 보안 설정

**파일**: `parantier-api/src/main/java/com/mapo/palantier/config/SecurityConfig.java`

```java
.authorizeHttpRequests(auth -> auth
    // Menu endpoints - 메뉴 조회는 인증 없이 접근 가능 (동적 네비게이션용)
    .requestMatchers("/api/menus/**").permitAll()
    // ...
)
```

- 메뉴 API는 `permitAll()`로 공개
- 서버 측에서 JWT 토큰을 확인하여 사용자 권한에 맞는 메뉴만 반환
- 비로그인 사용자는 공개 메뉴만 볼 수 있음

## 테스트 시나리오

### 시나리오 1: 로그인 후 메뉴 표시 확인
1. 비로그인 상태에서 헤더 확인 → 메뉴 없음
2. 로그인 (admin@example.com)
3. 즉시 헤더에 "대시보드", "관리자" 메뉴 표시됨 ✅
4. 새로고침 없이 메뉴 표시 확인 ✅

### 시나리오 2: 로그아웃 후 메뉴 숨김 확인
1. 로그인 상태에서 헤더에 메뉴 표시됨
2. 로그아웃 버튼 클릭
3. 즉시 헤더 메뉴가 사라짐 ✅
4. 새로고침 없이 메뉴 숨김 확인 ✅

### 시나리오 3: 권한별 메뉴 차이 확인
1. ADMIN으로 로그인 → "대시보드", "관리자" 메뉴 표시
2. 로그아웃
3. USER로 로그인 → "대시보드" 메뉴만 표시 (관리자 메뉴 없음)
