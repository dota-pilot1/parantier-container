# 트리 메뉴에서 인라인 입력으로 빠른 메뉴 추가 기능

## 날짜
2026-03-23

## 개요
메뉴 추가 시 전체 폼을 먼저 작성하지 않고, 트리에서 바로 메뉴 이름만 입력하여 빠르게 추가할 수 있는 인라인 입력 기능을 구현했습니다. 추가 후 선택적으로 상세 정보를 편집할 수 있습니다.

## 기존 문제점
- "하위 메뉴 추가" 클릭 시 → 우측 폼에서 모든 필드 입력 필요
- 메뉴 구조를 빠르게 구축하려면 반복적인 폼 작성이 번거로움
- 이름만 먼저 정하고 나중에 세부 설정을 하고 싶은 니즈

## 개선된 사용자 경험

### 이전 워크플로우
```
1. 부모 메뉴 우클릭 → "하위 메뉴 추가"
2. 우측 폼에 모든 필드 입력:
   - 메뉴명 ✓
   - 타입 ✓
   - 경로 ✓
   - 권한 ✓
   - 순서 ✓
   - 활성 상태 ✓
3. 저장 버튼 클릭
4. 다음 메뉴 추가를 위해 1-3 반복
```

### 개선된 워크플로우
```
1. 부모 메뉴 우클릭 → "하위 메뉴 추가"
2. 트리에서 바로 인라인 입력 필드 표시
3. 메뉴 이름만 입력하고 Enter
4. 즉시 생성 + 자동으로 상세 폼에 선택됨
5. (선택적) 필요한 경우에만 상세 정보 편집
```

**장점:**
- 빠른 메뉴 구조 구축: 이름만 연속으로 입력 가능
- 단계 감소: 필수 정보(이름)만 먼저, 선택 정보는 나중에
- 직관적: 트리에서 바로 추가되는 것을 시각적으로 확인

## UI/UX 구현 상세

### 1. 인라인 입력 컴포넌트 (InlineMenuInput.tsx)

**위치:** `/parantier-front/src/pages/admin/menus/components/InlineMenuInput.tsx`

**주요 기능:**
```tsx
interface InlineMenuInputProps {
  depth: number              // 트리 들여쓰기 레벨
  onSubmit: (name: string) => void  // Enter 시 호출
  onCancel: () => void       // Esc 또는 blur 시 호출
}
```

**UI 특징:**
- 트리 노드와 동일한 스타일/들여쓰기로 자연스럽게 통합
- 파란색 배경(bg-blue-50)으로 입력 중임을 명확히 표시
- 자동 포커스: 마운트 즉시 입력 가능
- 플레이스홀더: "메뉴 이름 입력 (Enter: 저장, Esc: 취소)"

**키보드 인터랙션:**
```tsx
const handleKeyDown = (e: React.KeyboardEvent) => {
  if (e.key === 'Enter') {
    e.preventDefault()
    handleSubmit()  // 이름이 비어있지 않으면 onSubmit 호출
  } else if (e.key === 'Escape') {
    e.preventDefault()
    onCancel()  // 입력 취소
  }
}
```

**자동 포커스 구현:**
```tsx
const inputRef = useRef<HTMLInputElement>(null)

useEffect(() => {
  inputRef.current?.focus()
}, [])
```

### 2. 상태 관리 (MenusPage.tsx)

**새로운 상태 추가:**
```tsx
const [addingChildToId, setAddingChildToId] = useState<number | null>(null)
```

- null: 인라인 입력 비활성 상태
- number: 해당 ID의 메뉴에 자식을 추가 중

**하위 메뉴 추가 핸들러 변경:**
```tsx
// 변경: 인라인 입력 표시
const handleAddChildMenu = () => {
  if (contextMenu) {
    // 부모 메뉴 자동 확장
    setExpandedIds((prev) => new Set(prev).add(contextMenu.menu.id))
    // 인라인 입력 모드 활성화
    setAddingChildToId(contextMenu.menu.id)
  }
}
```

**인라인 입력 제출 처리:**
```tsx
const handleInlineSubmit = (parentId: number, name: string) => {
  const requestData: CreateMenuRequest = {
    name,
    parentId,
    menuType: 'SUB',  // 기본값: 하위 메뉴는 SUB 타입
    orderNum: 0,
  }

  createMutation.mutate(requestData, {
    onSuccess: (newMenu) => {
      // ✅ 생성된 메뉴 자동 선택 (상세 편집 가능)
      setSelectedMenu(newMenu)
      setIsCreating(false)
      setParentMenu(null)
      setAddingChildToId(null)
    },
  })
}
```

**핵심 포인트:**
- menuType: 'SUB': 하위 메뉴는 기본적으로 SUB 타입
- orderNum: 0: 기본 순서값
- onSuccess 콜백에서 자동 선택: 생성 즉시 우측 폼에 표시되어 추가 편집 가능

### 3. 트리 노드 통합 (TreeNode.tsx)

**Props 추가:**
```tsx
interface TreeNodeProps {
  // ... 기존 props
  addingChildToId: number | null
  onInlineSubmit: (parentId: number, name: string) => void
  onInlineCancel: () => void
}
```

**인라인 입력 렌더링:**
```tsx
{/* 인라인 입력 (하위 메뉴 추가) */}
{isExpanded && addingChildToId === menu.id && (
  <InlineMenuInput
    depth={depth + 1}
    onSubmit={(name) => onInlineSubmit(menu.id, name)}
    onCancel={onInlineCancel}
  />
)}
```

**렌더링 조건:**
- isExpanded: 부모 메뉴가 펼쳐져 있어야 함
- addingChildToId === menu.id: 현재 메뉴에 자식을 추가 중일 때만

**깊이 전달:**
- depth={depth + 1}: 부모보다 한 단계 들여쓰기되어 표시

## 기본값 정책

인라인 입력으로 생성되는 메뉴의 기본값:

| 필드 | 기본값 | 이유 |
|------|--------|------|
| name | 사용자 입력 | 필수 필드 |
| parentId | 부모 메뉴 ID | 하위 메뉴 추가이므로 자동 설정 |
| menuType | 'SUB' | 하위 메뉴는 일반적으로 SUB 타입 |
| orderNum | 0 | 기본 순서, 나중에 조정 가능 |
| path | undefined | 선택적, 상세 편집에서 추가 |
| requiredRole | undefined | 선택적, 기본은 공개 |
| icon | undefined | 선택적 |

## 자동 선택 기능

**구현 이유:**
- 생성 직후 경로, 권한 등을 바로 편집하고 싶은 경우가 많음
- 메뉴가 제대로 생성됐는지 즉시 확인 가능

**구현 방법:**
```tsx
createMutation.mutate(requestData, {
  onSuccess: (newMenu) => {
    setSelectedMenu(newMenu)  // ← 자동 선택
    setIsCreating(false)
    setParentMenu(null)
    setAddingChildToId(null)
  },
})
```

## 결론

인라인 입력 기능으로 메뉴 관리의 사용성이 크게 개선되었습니다:

- **빠른 구조 구축**: 이름만 연속 입력하여 메뉴 트리 빠르게 생성
- **단계 감소**: 필수 정보만 먼저, 선택 정보는 나중에
- **직관적 UX**: 트리에서 바로 추가, 시각적 피드백 명확
- **유연성 유지**: 상세 편집이 필요하면 자동 선택된 폼에서 즉시 가능

핵심은 **단계 분리**입니다:
1. 빠른 생성 (이름만)
2. 선택적 편집 (상세 정보)

이로써 메뉴 관리 페이지의 3차 구현이 완료되었습니다.
