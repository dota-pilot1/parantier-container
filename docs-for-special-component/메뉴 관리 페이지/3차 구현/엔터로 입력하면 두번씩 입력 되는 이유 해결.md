# 엔터로 입력하면 두 번씩 입력되는 문제 해결

## 날짜
2026-03-23

## 문제 증상
- 인라인 입력에서 "취미 관리" 입력 후 Enter
- 결과: "취미 관리", "리" 두 개의 메뉴가 생성됨
- 예상: "취미 관리" 한 개만 생성되어야 함

## 근본 원인

### 초기 구현의 문제 (onBlur 사용)
```tsx
// 문제가 있던 코드
<input
  onKeyDown={handleKeyDown}
  onBlur={onCancel}  // ← Enter 후 자동으로 blur 발생
/>
```

**문제:**
- Enter 키를 누르면 브라우저가 자동으로 포커스를 이동시킴
- `onKeyDown` → `handleSubmit` 실행 중에 `onBlur` 이벤트 발생
- 타이밍 이슈로 상태가 꼬임

## 해결 방법 비교

### 방법 1: isSubmittingRef 플래그 사용 (복잡함)
```tsx
const isSubmittingRef = useRef(false)

const handleSubmit = () => {
  if (trimmed && !isSubmittingRef.current) {
    isSubmittingRef.current = true
    onSubmit(trimmed)
  }
}

const handleBlur = () => {
  if (!isSubmittingRef.current) {  // ← 제출 중 체크
    onCancel()
  }
}
```

**단점:**
- 직관적이지 않음
- 플래그 관리 필요
- onBlur와 onSubmit 간의 복잡한 타이밍 처리

### 방법 2: 외부 클릭 감지 (최종 채택) ✅
```tsx
export function InlineMenuInput({ depth, onSubmit, onCancel }: InlineMenuInputProps) {
  const containerRef = useRef<HTMLDivElement>(null)

  // 외부 클릭 감지
  useEffect(() => {
    const handleClickOutside = (e: MouseEvent) => {
      if (containerRef.current && !containerRef.current.contains(e.target as Node)) {
        onCancel()
      }
    }

    setTimeout(() => {
      document.addEventListener('mousedown', handleClickOutside)
    }, 0)

    return () => {
      document.removeEventListener('mousedown', handleClickOutside)
    }
  }, [onCancel])

  const handleSubmit = () => {
    const trimmed = value.trim()
    if (trimmed) {
      onSubmit(trimmed)  // 단순하게 제출만
    }
  }

  return (
    <div ref={containerRef}>
      <input
        onKeyDown={handleKeyDown}
        // onBlur 없음! ← 핵심
      />
    </div>
  )
}
```

**장점:**
- **직관적**: 외부 클릭하면 닫힘
- **단순**: 플래그나 복잡한 조건 불필요
- **명확**: Enter는 제출, 외부 클릭은 취소
- **표준 패턴**: 드롭다운, 모달 등에서 흔히 사용

## 핵심 변경 사항

### 1. onBlur 제거
```tsx
// Before
<input onBlur={onCancel} />

// After
<input />  // onBlur 없음
```

### 2. 외부 클릭 감지 추가
```tsx
const containerRef = useRef<HTMLDivElement>(null)

useEffect(() => {
  const handleClickOutside = (e: MouseEvent) => {
    // 컨테이너 외부 클릭 시 취소
    if (containerRef.current && !containerRef.current.contains(e.target as Node)) {
      onCancel()
    }
  }

  // setTimeout으로 다음 틱에 등록 (현재 클릭 이벤트와 충돌 방지)
  setTimeout(() => {
    document.addEventListener('mousedown', handleClickOutside)
  }, 0)

  return () => {
    document.removeEventListener('mousedown', handleClickOutside)
  }
}, [onCancel])
```

**왜 setTimeout을 사용하나?**
```
사용자가 "하위 메뉴 추가" 클릭
→ mousedown 이벤트 발생
→ InlineMenuInput 컴포넌트 마운트
→ useEffect 즉시 실행 → addEventListener
→ 같은 mousedown 이벤트가 handleClickOutside에서 감지됨!
→ 즉시 onCancel() 호출되어 입력 필드 닫힘 (의도하지 않음)

setTimeout 사용 시:
→ useEffect 실행 → setTimeout으로 다음 틱에 예약
→ 현재 mousedown 이벤트 완료
→ 다음 틱에 addEventListener 등록
→ 이후 클릭부터 감지 시작 ✅
```

### 3. containerRef로 영역 지정
```tsx
<div ref={containerRef}>
  <input />
</div>
```

**contains() 메서드:**
```tsx
containerRef.current.contains(e.target as Node)
// 클릭된 요소가 컨테이너 내부에 있는지 확인
// 내부 클릭: true → 아무 일도 안 함
// 외부 클릭: false → onCancel() 호출
```

## 왜 이 방법이 더 나은가?

### 1. 직관성
| 방법 | 복잡도 | 직관성 |
|------|--------|--------|
| onBlur + 플래그 | 높음 | 낮음 (타이밍 이슈 이해 필요) |
| 외부 클릭 감지 | 중간 | 높음 (외부 클릭 = 닫힘) |

### 2. 코드 가독성
```tsx
// ❌ 복잡한 방법
const isSubmittingRef = useRef(false)
if (!isSubmittingRef.current) { /* ... */ }

// ✅ 단순한 방법
if (!containerRef.current.contains(e.target)) { onCancel() }
```

### 3. 유지보수성
- 외부 클릭 패턴은 React 생태계에서 표준적으로 사용
- 다른 개발자가 보기에도 익숙한 패턴
- 버그 발생 가능성 낮음

### 4. 확장성
```tsx
// 외부 클릭 패턴은 다른 기능 추가에도 유리
const handleClickOutside = (e: MouseEvent) => {
  if (containerRef.current && !containerRef.current.contains(e.target)) {
    // 추가 로직 (예: 변경사항 저장 확인)
    if (value.trim() && confirm('입력 중인 내용을 저장하시겠습니까?')) {
      handleSubmit()
    } else {
      onCancel()
    }
  }
}
```

## 교훈

### 1. onBlur는 만능이 아니다
- onBlur는 **모든** 포커스 손실에 반응
- Enter, Tab, 외부 클릭 모두 구분 안 됨
- 세밀한 제어가 필요한 경우 부적합

### 2. 이벤트 타이밍 이슈는 복잡하다
- onKeyDown, onBlur, onClick 등이 복합적으로 작용
- 플래그로 해결 가능하지만 복잡도 증가
- 가능하면 단순한 패턴 선택

### 3. 표준 패턴을 따르자
- 외부 클릭 감지는 검증된 패턴
- 모달, 드롭다운, 팝업 등에서 널리 사용
- 직관적이고 유지보수하기 쉬움

## 결론

onBlur + 플래그 방식은 작동하지만 **직관적이지 않고 복잡**합니다.

**최종 해결책: 외부 클릭 감지**
- onBlur 제거
- mousedown 이벤트로 외부 클릭 감지
- containerRef.contains()로 영역 판단
- setTimeout으로 초기 클릭 충돌 방지

**핵심 원칙:**
- 단순하고 직관적인 패턴 선택
- 검증된 표준 패턴 활용
- 불필요한 복잡도 제거

이제 코드가 훨씬 더 읽기 쉽고 유지보수하기 좋아졌습니다!
