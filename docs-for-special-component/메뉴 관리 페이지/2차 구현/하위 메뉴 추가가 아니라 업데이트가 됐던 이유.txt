# 하위 메뉴 추가가 아니라 업데이트가 됐던 이유

## 날짜
2026-03-23

## 문제 증상
- "유저 관리" 우클릭 → "하위 메뉴 추가" 클릭
- 예상: 빈 폼에 "부모 메뉴: 유저 관리" 표시
- 실제: "유저 관리" 데이터가 그대로 폼에 표시됨
- 결과: 새 메뉴 추가가 아니라 기존 메뉴 업데이트가 실행됨

## 근본 원인

### React useState의 초기화 타이밍 문제
```tsx
// MenuEditForm.tsx (문제 코드)
export function MenuEditForm({ menu, parentMenu, onSave, onDelete, onCancel }: MenuEditFormProps) {
  const [formData, setFormData] = useState<Partial<Menu>>(
    menu || {
      name: '',
      path: '',
      menuType: 'SIDE' as MenuType,
      requiredRole: null,
      orderNum: 0,
      isActive: true,
      parentId: parentMenu?.id || null,  // ← 초기 렌더링 시에만 실행
    }
  )
```

**문제점:**
- `useState`의 초기값은 **컴포넌트 최초 마운트 시에만** 평가됨
- 이후 `menu`나 `parentMenu` props가 변경되어도 `formData`는 업데이트되지 않음
- 따라서 "유저 관리" 선택 후 → "하위 메뉴 추가" 클릭해도 기존 `formData`가 유지됨

## 실행 흐름 분석

### 1. 초기 상태 (유저 관리 선택)
```
사용자 클릭: "유저 관리" 선택
→ setSelectedMenu(유저 관리)
→ MenuEditForm 렌더링
→ useState 초기화: formData = { name: "유저 관리", id: 10, ... }
```

### 2. 하위 메뉴 추가 시도
```
사용자 우클릭: "유저 관리" → "하위 메뉴 추가"
→ handleAddChildMenu() 실행:
   - setParentMenu(유저 관리)
   - setSelectedMenu(null)
   - setIsCreating(true)
→ MenuEditForm 리렌더링
→ BUT! useState는 다시 실행되지 않음
→ formData는 여전히 { name: "유저 관리", id: 10, ... }
```

### 3. 잘못된 동작
```
사용자 입력: "권한 관리" 입력 후 저장
→ formData.id가 10으로 존재
→ handleSave()에서 if (menuData.id) 조건 충족
→ updateMutation 실행 (생성이 아닌 수정!)
→ 결과: "유저 관리" 메뉴가 "권한 관리"로 이름 변경됨
```

## 해결 방법

### useEffect를 사용한 props 동기화
```tsx
// MenuEditForm.tsx (수정 후)
export function MenuEditForm({ menu, parentMenu, onSave, onDelete, onCancel }: MenuEditFormProps) {
  const [formData, setFormData] = useState<Partial<Menu>>(
    menu || {
      name: '',
      path: '',
      menuType: 'SIDE' as MenuType,
      requiredRole: null,
      orderNum: 0,
      isActive: true,
      parentId: parentMenu?.id || null,
    }
  )

  // menu나 parentMenu가 변경되면 formData 초기화
  useEffect(() => {
    setFormData(
      menu || {
        name: '',
        path: '',
        menuType: 'SIDE' as MenuType,
        requiredRole: null,
        orderNum: 0,
        isActive: true,
        parentId: parentMenu?.id || null,
      }
    )
  }, [menu, parentMenu])  // ← 의존성 배열에 menu, parentMenu 추가
```

### 동작 원리
1. `menu` 또는 `parentMenu` props가 변경될 때마다 useEffect 실행
2. `setFormData()`로 상태 강제 업데이트
3. `menu === null`이면 빈 폼 데이터로 초기화
4. `parentMenu?.id`를 `parentId`에 자동 설정

## 수정 후 정상 흐름

### 하위 메뉴 추가 시나리오
```
사용자 우클릭: "유저 관리" → "하위 메뉴 추가"
→ handleAddChildMenu() 실행:
   - setParentMenu(유저 관리)
   - setSelectedMenu(null)      ← menu = null
   - setIsCreating(true)
→ useEffect 감지: menu가 null로 변경됨
→ setFormData({ name: '', ..., parentId: 10 })  ← 빈 폼으로 초기화
→ 화면: 빈 입력 폼 + "부모 메뉴: 유저 관리" 표시
→ 사용자 입력: "권한 관리"
→ 저장: formData.id가 없음
→ createMutation 실행 ✓
→ 결과: "유저 관리" 아래 "권한 관리" 새로 생성됨
```

## 교훈

### React Hooks 사용 시 주의사항
1. **useState 초기값은 최초 1회만 평가됨**
   - Props가 변경되어도 자동으로 반영되지 않음

2. **Props와 State 동기화가 필요한 경우:**
   - useEffect + 의존성 배열 사용
   - 또는 key prop으로 컴포넌트 강제 리마운트

3. **대안: key prop 사용**
   ```tsx
   <MenuEditForm
     key={`${selectedMenu?.id}-${parentMenu?.id}`}
     menu={selectedMenu}
     parentMenu={parentMenu}
     ...
   />
   ```
   - key가 변경되면 컴포넌트가 완전히 새로 마운트됨
   - useState가 처음부터 다시 실행됨
   - 하지만 useEffect 방식이 더 명확하고 예측 가능함
