# 메뉴 관리 페이지: 엔터키 검색 기능 구현

## 개요

메뉴 관리 페이지(MenusPage)의 검색 입력창에서 엔터키를 누르면 검색을 확정하고 포커스를 해제하는 기능을 추가했습니다.

## 변경 일시

- **일시**: 2026-03-23
- **파일**: `parantier-front/src/pages/admin/menus/MenusPage.tsx`

## 구현 내용

### 1. 검색 동작 방식

현재 MenusPage의 검색 기능은 **실시간 필터링** 방식으로 동작합니다:

- 사용자가 입력창에 타이핑하는 즉시 `onChange` 이벤트로 검색어가 업데이트됨
- `searchQuery` 상태가 변경되면 자동으로 메뉴 트리가 필터링됨
- 매칭된 메뉴의 부모 노드가 자동으로 확장됨

### 2. 엔터키 처리 추가

엔터키를 눌렀을 때 검색을 확정하는 UX를 추가했습니다.

**파일**: `parantier-front/src/pages/admin/menus/MenusPage.tsx` (Line 397-425)

```tsx
{/* 검색 입력창과 추가 버튼 */}
<div className="flex gap-2">
  <div className="flex-1 relative">
    <input
      type="text"
      value={searchQuery}
      onChange={(e) => setSearchQuery(e.target.value)}
      onKeyDown={(e) => {
        if (e.key === 'Enter') {
          // 엔터키 입력 시 검색 실행 (이미 onChange로 실시간 필터링 중)
          e.currentTarget.blur() // 포커스 해제하여 검색 완료 표시
        }
      }}
      placeholder="메뉴 검색 (Enter로 확인)..."
      className="w-full px-3 py-2 text-sm border border-gray-300 rounded-md focus:outline-none focus:ring-2 focus:ring-blue-500"
    />
    {searchQuery && (
      <button
        onClick={() => setSearchQuery('')}
        className="absolute right-2 top-1/2 -translate-y-1/2 text-gray-400 hover:text-gray-600"
      >
        ✕
      </button>
    )}
  </div>
  <Button onClick={handleCreateNew} size="sm" className="flex-shrink-0">
    +
  </Button>
</div>
```

### 3. 주요 변경 사항

#### 3.1 `onKeyDown` 이벤트 핸들러 추가

```tsx
onKeyDown={(e) => {
  if (e.key === 'Enter') {
    // 엔터키 입력 시 검색 실행 (이미 onChange로 실시간 필터링 중)
    e.currentTarget.blur() // 포커스 해제하여 검색 완료 표시
  }
}}
```

**동작 설명**:
1. 사용자가 엔터키를 누름
2. 이미 `onChange`로 실시간 필터링이 적용된 상태
3. `blur()`를 호출하여 입력창의 포커스를 해제
4. 포커스 해제로 검색이 완료되었음을 시각적으로 표시

#### 3.2 Placeholder 텍스트 개선

**변경 전**:
```tsx
placeholder="메뉴 검색..."
```

**변경 후**:
```tsx
placeholder="메뉴 검색 (Enter로 확인)..."
```

사용자에게 엔터키로 검색을 확정할 수 있다는 힌트를 제공합니다.

## 실시간 필터링 로직

메뉴 검색 기능은 다음과 같이 동작합니다:

### 1. 검색어 매칭

```tsx
const getFilteredMenusAndExpandedIds = (): {
  filteredMenus: Menu[]
  highlightedIds: Set<number>
  autoExpandedIds: Set<number>
} => {
  if (!searchQuery.trim()) {
    return {
      filteredMenus: menus,
      highlightedIds: new Set(),
      autoExpandedIds: new Set(),
    }
  }

  const query = searchQuery.toLowerCase()
  const flattenMenus = (menuList: Menu[]): Menu[] => {
    const result: Menu[] = []
    menuList.forEach((menu) => {
      result.push(menu)
      if (menu.children && menu.children.length > 0) {
        result.push(...flattenMenus(menu.children))
      }
    })
    return result
  }

  const allMenus = flattenMenus(menus)

  // 매칭된 메뉴 찾기
  const matchedMenus = allMenus.filter((menu) =>
    menu.name.toLowerCase().includes(query)
  )

  const highlightedIds = new Set(matchedMenus.map((m) => m.id))

  // 매칭된 메뉴의 모든 부모 ID 수집
  const getParentIds = (menuId: number | null): number[] => {
    if (!menuId) return []
    const parent = allMenus.find((m) => m.id === menuId)
    if (!parent || !parent.parentId) return []
    return [parent.parentId, ...getParentIds(parent.parentId)]
  }

  const autoExpandedIds = new Set<number>()
  matchedMenus.forEach((menu) => {
    const parentIds = getParentIds(menu.id)
    parentIds.forEach((id) => autoExpandedIds.add(id))
  })

  return {
    filteredMenus: menus,
    highlightedIds,
    autoExpandedIds,
  }
}
```

### 2. 자동 확장 적용

```tsx
// 검색어가 있을 때 자동 확장 적용
useEffect(() => {
  if (searchQuery.trim() && autoExpandedIds.size > 0) {
    setExpandedIds(autoExpandedIds)
  }
}, [searchQuery, autoExpandedIds])
```

### 3. 하이라이트 표시

매칭된 메뉴는 `highlightedIds` Set에 저장되어 TreeView 컴포넌트에서 시각적으로 강조 표시됩니다.

## 키보드 인터랙션

| 키 | 동작 |
|---|---|
| **Enter** | 검색 확정 (포커스 해제) |
| **ESC** | 컨텍스트 메뉴 닫기 (검색창 포커스 중에는 영향 없음) |
| **타이핑** | 실시간 필터링 |

## 검색 결과 표시

검색어가 입력되면 하단에 매칭된 메뉴 개수가 표시됩니다:

```tsx
{/* 검색 결과 카운트 */}
{searchQuery && (
  <div className="mt-2 text-xs text-gray-600">
    {highlightedIds.size > 0
      ? `${highlightedIds.size}개의 메뉴 찾음`
      : '검색 결과 없음'}
  </div>
)}
```

**예시**:
- 검색어 입력: "관리" → `2개의 메뉴 찾음`
- 검색어 입력: "xyz123" → `검색 결과 없음`

## 장점

1. **실시간 피드백**: 타이핑하는 즉시 결과를 확인 가능
2. **명확한 완료 표시**: 엔터키로 검색 완료를 명시적으로 표현
3. **자동 확장**: 매칭된 메뉴의 부모 노드가 자동으로 펼쳐져 찾기 쉬움
4. **시각적 하이라이트**: 검색 결과가 강조 표시되어 가독성 향상
5. **빠른 초기화**: ✕ 버튼으로 검색어를 쉽게 지울 수 있음

## 기술적 세부사항

### React 상태 관리

- `searchQuery`: 검색어 상태 (useState)
- `highlightedIds`: 매칭된 메뉴 ID Set
- `autoExpandedIds`: 자동 확장할 부모 노드 ID Set
- `expandedIds`: 현재 확장된 노드 ID Set

### 이벤트 처리

- `onChange`: 입력 즉시 검색어 업데이트
- `onKeyDown`: 엔터키 감지 및 포커스 해제
- `onClick` (✕ 버튼): 검색어 초기화

### 성능 최적화

- `useMemo`로 검색 로직 메모이제이션 (필요시 추가 가능)
- `useEffect`로 자동 확장 로직 분리
- Set 자료구조로 빠른 조회 성능

## 향후 개선 가능 사항

1. **디바운싱**: 타이핑 중 과도한 렌더링 방지
2. **정규식 검색**: 고급 검색 패턴 지원
3. **검색 히스토리**: 최근 검색어 저장
4. **필터 옵션**: 메뉴 타입별 필터링 (HEADER, SIDE, SUB)
5. **키보드 내비게이션**: 화살표 키로 검색 결과 이동

## 참고 사항

- 현재 검색은 **메뉴 이름(name)** 필드만 대상으로 함
- 대소문자 구분 없이 검색 (toLowerCase 처리)
- 부분 일치 검색 지원 (includes 메서드)
- 검색어가 비어있으면 전체 메뉴 트리 표시
